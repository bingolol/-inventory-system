# 业务因果链

> 记录全部业务场景的因果依赖关系,按"业务事实→会计实务→理应数据链→关键因果"结构。
> 更新日期: 2026-07-05

---

## A. 销售类业务(5 种现实场景)

### A1. 散客销售,当场收款,不开发票

**业务事实**:客户上门买货,付钱走人,没要发票。

**会计实务处理**:
- 出库:库存减少,确认出库成本
- 收款:银行/现金增加
- 收入确认:按收到的钱确认收入(小规模直接按含税额确认收入)
- 税务:未开票收入仍需申报销项税(小规模按征收率分离)

```
出库 + 收款(外部输入)
  → 库存减少(因:出库)
  → 营业收入确认(因:收款,不依赖发票)
  → 销项税确认(因:收款,按征收率分离)
  → 银行/现金增加(因:收款)
```

**关键因果**:不开发票 ≠ 不确认收入 ≠ 不交税。收入的因是"交易完成收到钱",发票只是税务凭据之一。

### A2. 散客销售,当场收款,开发票

**业务事实**:客户上门买货付钱,要了发票。

**会计实务处理**:
- 出库:库存减少,确认出库成本
- 收款:银行/现金增加
- 开票:确认收入 + 销项税
- 收入和税额的确认以发票为准

```
出库 + 收款 + 开票(三个外部输入)
  → 库存减少(因:出库)
  → 营业收入确认(因:发票不含税额)
  → 销项税确认(因:发票税额)
  → 银行/现金增加(因:收款)
```

**关键因果**:发票是收入和销项税的真相源,但收款是独立的资金流,三者可以不同步(虽然现场销售通常同步)。

### A3. 单位客户,先发货后收款(账期),开发票

**业务事实**:客户是公司,签合同先发货,约定 30 天后付款,发货时开发票。

**会计实务处理**((《小企业会计准则》第五十九条)):
- 发货时:出库 + 确认收入 + 确认销项税 + 确认应收账款
- 30 天后收款:应收账款减少 + 银行存款增加(不重复确认收入)

```
发货 + 开票(外部输入)
  → 库存减少
  → 营业收入确认(因:发货 + 发票)
  → 销项税确认(因:发票)
  → 应收账款增加(因:发货未收款)

30天后收款(外部输入)
  → 应收账款减少
  → 银行存款增加
  (不触及收入和税,因为发货时已确认)
```

**关键因果**:收入和税在"发货+开票"时一次性确认,账期收款只影响资金流和应收账款,不再触及收入。这就是"权责发生制"——权利义务发生即确认,不管钱到没到。

### A4. 单位客户,先收款后发货(预收),开发票

**业务事实**:客户预付全款,企业 1 个月后才发货,收款时开发票。

**会计实务处理**(《小企业会计准则》第五十九条:预收款方式,在发出商品时确认收入):
- 收款时:银行存款增加 + 预收账款增加(不确认收入)
- 发货时:出库 + 预收账款转收入 + 确认销项税

```
预收款(外部输入)
  → 银行存款增加
  → 预收账款增加(负债,不是收入)
  (不确认收入,不确认销项税)

发货 + 开票(外部输入)
  → 库存减少
  → 预收账款转出 → 营业收入确认
  → 销项税确认
```

**关键因果**:预收的钱是负债(欠客户的货),不是收入。收入在发货时才确认。钱先到 ≠ 收入先确认。

### A5. 分期收款销售

**业务事实**:客户分 3 期付款,每期 30 天,发货时开发票。

**会计实务处理**(《小企业会计准则》第六十条:分期收款方式,在合同约定收款日期确认收入):
- 发货时:出库 + 按合同约定分期确认应收账款(全部应收)
- 每期收款日:按该期金额确认收入 + 销项税
- **注意**:小企业准则规定按合同约定收款日确认收入,不按发货日

```
发货(外部输入)
  → 库存减少
  → 应收账款增加(全部合同金额,但不立即确认收入)

第1期收款日 + 开票(外部输入)
  → 确认本期收入 + 销项税
  → 应收账款减少(本期部分)

第2、3期同上
```

**关键因果**:收入不是在发货时一次确认,而是按合同约定收款日分期确认。这是小企业准则的特殊规定。

## B. 采购类业务(3 种现实场景)

### B1. 正常采购,货到票到

**业务事实**:进货,货到同时收到发票,付款。

**会计实务处理**:
- 入库:库存增加 + 进项税确认(一般纳税人专票) + 应付账款增加
- 付款:应付账款减少 + 银行存款减少

```
入库 + 发票(外部输入)
  → 库存增加(因:入库,按不含税单价,一般纳税人专票)
  → 进项税确认(因:已认证专票)
  → 应付账款增加(因:未付款)

付款(外部输入)
  → 应付账款减少
  → 银行存款减少
```

### B2. 货到票未到(暂估入库)

**业务事实**:货到了,发票还没到(供应商下月才开),但本月已经用了这批货卖出去。

**会计实务处理**(实务惯例:月末暂估,次月冲回):
- 本月末:按合同价或估计价暂估入库(借:库存商品 贷:应付账款-暂估)
- 暂估时不确认进项税(没发票)
- 次月初:红字冲回暂估
- 次月发票到:按实际发票金额正常入账

```
货到(外部输入)
  → 暂估入库(因:货到了但没发票,按估计价)
  → 应付账款-暂估增加(因:暂估)

次月初冲回(系统动作)
  → 红字冲销暂估

发票到(外部输入)
  → 正式入库(因:实际发票)
  → 进项税确认(因:专票认证)
  → 应付账款增加(因:发票金额)
```

**关键因果**:暂估是"没有发票时的权宜之计",本质是估计值。发票到了必须冲回暂估,按实际金额重新入账。暂估成本不是真相源,发票才是。

### B3. 票到货未到(在途物资)

**业务事实**:发票先到,货还在路上。

**会计实务处理**:
- 发票到:确认进项税 + 在途物资(资产类科目)
- 货到:在途物资转库存商品

```
发票到(外部输入)
  → 在途物资确认(因:发票到了货没到)
  → 进项税确认(因:专票认证)

货到(外部输入)
  → 在途物资转出 → 库存商品增加
```

## C. 退货业务(2 种)

### C1. 销售退货

**会计实务处理**:
- 开红字发票(负数发票)
- 库存增加(货退回来)
- 冲减收入和销项税
- 冲减营业成本

```
退货 + 红字发票(外部输入)
  → 库存增加(因:货退回)
  → 红字冲减收入(因:红字发票)
  → 红字冲减销项税(因:红字发票)
  → 红字冲减营业成本(因:库存增加)
```

**关键因果**:退货必须开红字发票,否则季度报税时销项税虚高。红字发票是退货税务处理的真相源。

### C2. 采购退货

**会计实务处理**:
- 库存减少(货退给供应商)
- 冲减进项税(或进项税转出)
- 冲减应付账款或收款

```
退货(外部输入)
  → 库存减少(因:货退回给供应商)
  → 红字冲减进项税(因:退货)
  → 应付账款减少 或 银行存款增加(因:退款)
```

**关键因果**:退货成本必须用原入账成本冲减,不能用当前移动加权平均成本。

## D. 折扣与折让(3 种)

### D1. 商业折扣(打折销售)

**业务事实**:标价 100,打 8 折,实际卖 80。

**会计实务处理**:直接按折扣后金额确认收入和税。发票按折扣后金额开。

```
销售 + 商业折扣(外部输入)
  → 营业收入 = 折扣后金额(不含税)
  → 销项税 = 折扣后金额 × 税率
```

**关键因果**:商业折扣直接体现在发票金额里,不需要单独科目处理。

### D2. 现金折扣(鼓励早付款)

**业务事实**:账期 30 天,客户 10 天内付款给 2% 折扣。

**会计实务处理**(总额法):
- 销售时:按全额确认收入和应收账款
- 客户提前付款享受折扣:折扣部分计入"财务费用"

```
销售(外部输入)
  → 营业收入 = 全额(不含税)
  → 应收账款 = 全额(含税)

提前付款 + 折扣(外部输入)
  → 银行存款 = 实际收到金额
  → 财务费用 = 折扣金额(因:客户提前付款)
  → 应收账款减少 = 全额
```

**关键因果**:现金折扣不影响收入,只计入财务费用。收入的因是销售发票,折扣的因是提前付款行为。

### D3. 销售折让(质量问题让价)

**业务事实**:客户收到货发现部分质量问题,协商让价 10%。

**会计实务处理**:开红字发票冲减部分收入和销项税,不退回货物。

```
折让 + 红字发票(外部输入)
  → 红字冲减收入(因:红字发票)
  → 红字冲减销项税(因:红字发票)
  → 应收账款减少 或 银行存款退款
```

## E. 职工薪酬(工资)

**会计实务处理**:
- 计提时:管理费用/销售费用 + 应付职工薪酬(负债)
- 发放时:应付职工薪酬减少 + 银行存款减少 + 代扣个税(应交税费-个人所得税)

```
计提工资(外部输入:工资金额)
  → 管理费用/销售费用增加(因:计提)
  → 应付职工薪酬增加(因:计提,负债)

发放工资(外部输入:实发金额、代扣个税)
  → 应付职工薪酬减少
  → 银行存款减少(实发金额)
  → 应交税费-个人所得税增加(因:代扣)
```

**关键因果**:工资费用在"计提"时确认,不在"发放"时确认。计提是因,发放是果。个税是代扣代缴,实发时才产生纳税义务。

## F. 固定资产(3 个环节)

### F1. 固定资产采购

**会计实务处理**:
- 一般纳税人专票:按不含税额计入固定资产原值,进项税抵扣
- 小规模/普票:按价税合计计入固定资产原值

```
采购 + 发票(外部输入)
  → 固定资产原值确认(因:发票不含税额或价税合计)
  → 进项税确认(因:一般纳税人专票)
  → 应付账款增加
```

### F2. 固定资产折旧

**会计实务处理**(《小企业会计准则》第三十一条:当月增加,下月提折旧):
- 月折旧额 = 原值 × (1 - 残值率) ÷ 使用年限(年限平均法)
- 累计折旧上限 = 原值 × (1 - 残值率)

```
固定资产原值(外部输入,来自采购)
+ 残值率、使用年限(政策配置)
+ "下月才提"规则(政策)
  → 月折旧额(计算)
  → 管理费用增加(因:折旧)
  → 累计折旧增加(因:折旧)
```

### F3. 固定资产处置

**会计实务处理**:
- 固定资产净值转清理:原值 - 累计折旧
- 收到处置款:银行存款增加 + 投资活动现金流入
- 净值 vs 处置款:差额计入营业外收支

```
处置决定 + 处置价(外部输入)
  → 固定资产原值转出
  → 累计折旧转出
  → 银行存款增加(因:收到处置款)
  → 投资活动现金流入(因:处置固定资产属于投资活动)
  → 处置价 vs 净值:
      价 > 净值 → 营业外收入
      价 < 净值 → 营业外支出
```

**关键因果**:处置固定资产的现金流必须归"投资活动",不能归"经营活动"。

## G. 无形资产

### G1. 无形资产采购

```
采购 + 发票(外部输入)
  → 无形资产原值确认(价税分离规则同固定资产)
  → 进项税确认(一般纳税人专票)
  → 应付账款增加
```

### G2. 无形资产摊销

**会计实务处理**(与固定资产折旧相反!):
- 当月增加的无形资产,当月开始摊销
- 处置当月不再摊销

```
无形资产原值 + 摊销年限(外部输入/政策)
+ "当月增当月摊"规则(政策)
  → 月摊销额 = 原值 ÷ 摊销年限(一般无残值)
  → 管理费用增加
  → 累计摊销增加
```

**关键因果**:无形资产"当月增当月摊",固定资产"当月增下月提"。两者规则相反,不能混淆。

## H. 费用(3 种)

### H1. 费用发生并付款

```
费用单据(外部输入:房租发票、水电费等)
  → 期间费用确认(管理/销售/财务费用,因:费用单据)
  → 银行存款减少(因:付款)
```

### H2. 费用发生未付款(应付)

```
费用单据(外部输入)
  → 期间费用确认(因:费用单据)
  → 应付账款增加(因:未付款)

付款(外部输入)
  → 应付账款减少
  → 银行存款减少
```

### H3. 个人垫付费用

**业务事实**:员工先垫付,公司后还。

```
费用发生 + 员工垫付(外部输入)
  → 期间费用确认(因:费用单据)
  → 其他应付款增加(因:欠员工钱)

还款(外部输入)
  → 其他应付款减少
  → 银行存款减少
```

## I. 资产减值(小企业准则特殊)

**会计实务处理**(《小企业会计准则》第六条:资产按照成本计量,不计提减值准备):
- 小企业不计提存货跌价准备、固定资产减值准备、无形资产减值准备、坏账准备
- 资产按成本计量,实际损失发生时直接核销

```
(无减值计提动作)

实际发生坏账/损失(外部输入)
  → 营业外支出增加(因:实际损失)
  → 应收账款/资产核销
```

**关键因果**:小企业不预提减值,损失实际发生时才入账。这与企业会计准则不同。

### J1. 营业外收入

```
营业外收入事件(外部输入)
  → 营业外收入确认(因:事件发生)

```

### J2. 营业外支出

**会计实务**:罚款支出、捐赠支出、非常损失、坏账实际核销

```
营业外支出事件(外部输入)
  → 营业外支出确认
  → 银行存款减少 或 资产核销
```

## K. 视同销售(8 种情形,小企业常见 2 种)

### K1. 自产货物用于集体福利

**业务事实**:把自己生产的商品发给员工当福利。

**会计实务处理**(视同销售):
- 按公允价值(市场价)确认收入 + 销项税
- 同时按成本结转营业成本
- 借:应付职工薪酬 / 贷:主营业务收入 + 销项税
- 借:主营业务成本 / 贷:库存商品

```
发福利决定(外部输入)
  → 库存减少(因:出库)
  → 营业收入确认(因:视同销售,按公允价)
  → 销项税确认(因:视同销售,按公允价×税率)
  → 营业成本确认(因:出库成本)
  → 应付职工薪酬减少(因:用货抵福利)
```

### K2. 货物无偿赠送

**会计实务处理**:按公允价值确认收入 + 销项税,结转成本。借方是"营业外支出"。

```
同 K1,但借方是"营业外支出"而不是"应付职工薪酬"
```

## L. 税务申报(三税种)

### L1. 增值税申报

**会计实务**:
- 销项税额 = 所有销售发票税额(含红字冲减)+ 视同销售计算的税额
- 进项税额 = 所有已认证进项专票税额 - 进项税转出(用于福利等)
- 应纳增值税 = 销项 - 进项(留抵下期)

```
所有销项发票 + 视同销售(外部输入)
  → 销项税额合计

所有已认证进项专票(外部输入) + 进项税转出(因:福利等)
  → 进项税额合计

销项 - 进项 → 应纳增值税
  → 增值税申报表
```

**关键因果**:增值税的根因是发票 + 视同销售事件,不是账本科目余额。

### L2. 附加税申报

**会计实务**:
- 附加税 = 实际缴纳的增值税 × 附加税率
- 附加税率:城建税(市区 7%)、教育费附加 3%、地方教育附加 2%,合计 12%
- 附加税减半:6%（独立于所得税小微判定，读取 `Account.surcharge_halved` 字段，BR-25）
- 教育费附加/地方教育附加季度免征：季度销售额 ≤ 30 万时免征（财税〔2016〕12 号），城建税不享受此免征

```
应纳增值税(衍生自发票)
× 附加税率(政策)
  → 附加税额
  → 附加税申报表

入账:
  → 税金及附加增加(借 6403)
  → 应交附加税增加(贷 222104)
```

**关键因果**:附加税的因是增值税,增值税的因是发票。附加税是"衍生的衍生"。

### L3. 所得税申报

**会计实务**:
- 利润总额 = 收入 - 成本 - 期间费用(含附加税) + 营业外收入 - 营业外支出
- 应纳税所得额 = 利润总额 ± 纳税调整(小企业通常无调整)
- 所得税 = 应纳税所得额 × 税率(小型微利 5%,普通 25%,个人实体 0%)

```
营业收入(衍生自销售发票)
- 营业成本(衍生自出库成本)
- 期间费用(衍生自费用单据 + 折旧 + 摊销 + 附加税)
+ 营业外收入(衍生自营业外事件)
- 营业外支出(衍生自营业外事件)
  → 利润总额

利润总额 × 所得税率(政策,含小型微利优惠)
  → 所得税额
  → 所得税申报表

入账:
  → 所得税费用增加(借 6801)
  → 应交所得税增加(贷 222105)
```

**关键因果**:所得税是"衍生的衍生的衍生"——它的因是利润,利润的因是收入和成本费用,收入和成本费用的因是发票和单据。

## M. 银行对账

### M1. 银行对账的核心

**会计实务**:银行对账 = 账面银行存款余额 vs 银行对账单余额,差额通过"银行存款余额调节表"调整。

常见差异:
- 银行已收企业未收(银行流水有,账上没有)
- 银行已付企业未付(如银行扣手续费,账上还没记)
- 企业已收银行未收(如存入支票,银行还没到账)
- 企业已付银行未付(如开出支票,持票人还没兑现)

```
账面银行存款余额(衍生自所有银行相关凭证)
vs
银行对账单余额(外部输入)
  → 差额
  → 余额调节表(调整项:未达账项)
  → 调节后余额应相等
```

**关键因果**:银行流水的因是实际资金进出(外部事件),账面银行存款的因是会计凭证。两者必须最终一致,差异只是时间差(未达账项)。

## N. 长期待摊费用

### N1. 租入固定资产改良支出

**业务事实**:租的门面装修,装修费按租期摊销。

**会计实务处理**:
- 发生装修费:计入长期待摊费用(资产类)
- 按租期平均摊销:每期管理费用

```
装修费发票(外部输入)
  → 长期待摊费用确认(因:资本化)
  → 银行存款减少(因:付款)

每期摊销(政策:按租期)
  → 管理费用增加(因:摊销)
  → 长期待摊费用减少
```

## O. 报表

### O1. 资产负债表

**会计实务**:资产 = 负债 + 所有者权益(恒等式)

```
所有资产类科目余额(衍生自凭证)
  → 资产总额

所有负债类科目余额(衍生自凭证)
  → 负债总额

所有权益类科目余额(衍生自凭证)
  → 权益总额

资产总额 == 负债总额 + 权益总额 (必须平衡)
```

**关键因果**:报表所有数字都是"果",它们的"因"是凭证,凭证的"因"是外部输入。报表不能反过来当源。

### O2. 利润表

```
营业收入(衍生自销售发票) - 营业成本(衍生自出库) - 期间费用(衍生自费用+折旧+摊销+附加税) + 营业外收入 - 营业外支出
  → 利润总额
利润总额 - 所得税
  → 净利润
```

## 三税种关系图

```
                  in_invoice(销项) ──→ 销项税额 ──┐
                                                   ↓
                              应纳增值税 ──────────┼──→ derived_vat_declaration
                                                   │
                  in_invoice(进项认证专票) ──→ 进项税额
                                                   │
                                                   ↓
                              附加税 = 应纳增值税 × 附加税率 ──→ derived_surcharge_declaration
                                                   │
                                                   ↓
                              入账 6403 / 222104
                                                   │
                                                   ↓
                              利润 = 收入 - 成本 - 费用（含附加税）+ 营外 - 营外支出
                                                   │
                                                   ↓
                              所得税 = 利润 × 所得税率 ──→ derived_income_tax_declaration
                                                   │
                                                   ↓
                              入账 6801 / 222105
```

### 关键因果总结

| 税种 | 因 | 因的类型 |
|------|----|----------|
| 增值税 | 发票 + 视同销售事件 | 外部输入 |
| 附加税 | 增值税 | 衍生数字 |
| 所得税 | 利润 → 收入(发票) - 成本(出库) - 费用(费用单) | 衍生数字的衍生数字 |

### 核心原则

> 所有报表数字都是"果",它们的"因"是凭证,凭证的"因"是外部输入。报表不能反过来当源。
> 不开发票 ≠ 不确认收入 ≠ 不交税。
> 钱先到 ≠ 收入先确认(预收)。货先发 ≠ 收入先确认(分期收款)。
