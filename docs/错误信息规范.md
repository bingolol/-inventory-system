# 错误信息规范

## 核心原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **单一错误系统** | 全系统统一使用 `errors.py` 的 `BusinessError`（含 `ErrorCode` enum），禁止自建错误类 |
| 2 | **中文消息 + 模板化** | 消息全中文，通过 `ERROR_REGISTRY` 模板 + `data` 参数生成，禁止硬编码拼接 |
| 3 | **语义化 ErrorCode** | 不同错误场景不同 code，禁止一律 `VALIDATION_ERROR` |
| 4 | **禁止 `message=str(e)`** | 永远不要传 raw Python exception 给用户 |
| 5 | **结构化响应** | 响应必须包含完整 6 字段：`code` / `message` / `action` / `action_data` / `data` / `ai_instruction` |

## 响应格式

```json
{
  "error": {
    "code": "PRODUCT_NOT_FOUND",
    "message": "商品不存在: ID=42",
    "action": "none",
    "action_data": {},
    "data": {"product_id": 42},
    "ai_instruction": "STOP_RETRYING. 商品不存在，请检查商品ID或名称是否正确..."
  }
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `code` | string | ✅ | ErrorCode enum 值（大写蛇形） |
| `message` | string | ✅ | 中文消息，终端用户可读 |
| `action` | string | ✅ | `none`/`retry`/`user_confirm`/`user_input`/`user_select`/`login`/`contact_admin` |
| `action_data` | object | ✅ | action 的附加数据（确认按钮文本、选项列表等） |
| `data` | object | ✅ | 与错误关联的业务数据（ID、金额、数量等） |
| `ai_instruction` | string | ✅ | 对 AI Agent 的引导说明 |

## ErrorCode 使用规范

### 按领域分组

| 分组 | code 前缀 | 示例 |
|------|-----------|------|
| 库存 | `INVENTORY_*` | `INVENTORY_INSUFFICIENT` |
| 订单 | `ORDER_*` | `ORDER_NOT_FOUND` |
| 发票 | `INVOICE_*` | `INVOICE_DUPLICATE_NUMBER` |
| 财务 | `BALANCE_*`, `CASH_FLOW_*`, `INCOME_STATEMENT_*` | `BALANCE_SHEET_UNBALANCED` |
| 商品 | `PRODUCT_*` | `PRODUCT_HAS_TRANSACTIONS` |
| 合作伙伴 | `SUPPLIER_*`, `CUSTOMER_*` | `CUSTOMER_HAS_ORDERS` |
| 现金流 | `CASH_FLOW_*` | `CASH_FLOW_NOT_FOUND` |
| 费用 | `EXPENSE_*` | `EXPENSE_NOT_FOUND` |
| 固定资产 | `FIXED_ASSET_*` | `FIXED_ASSET_NOT_FOUND` |
| 银行账户 | `BANK_ACCOUNT_*` | `BANK_ACCOUNT_NOT_FOUND` |
| 通用 | 无前缀 | `VALIDATION_ERROR`, `DUPLICATE_ENTRY` |

### 选择原则

```
找到了具体的领域实体 → 用领域 ErrorCode（PRODUCT_NOT_FOUND ✓）
仅做校验不涉及实体 → 用 VALIDATION_ERROR（带 details data）
数据重复 → 用 DUPLICATE_ENTRY（带 details data）
不应该发生的错误 → 用 INTERNAL_ERROR（同时带 context data）
```

## 抛出错误的正确姿势

### ✅ 正确：模板化（优先）

```python
raise BusinessError(
    code=ErrorCode.PRODUCT_NOT_FOUND,
    data={"product_id": product_id}
)
```

消息从 `ERROR_REGISTRY` 模板 `"商品不存在: ID={product_id}"` 自动生成。

### ✅ 正确：模板化 + 覆盖消息

```python
raise BusinessError(
    code=ErrorCode.VALIDATION_ERROR,
    data={"details": "金额不能为负"},
)
```

### ❌ 错误：message=str(e)

```python
# 反模式 - 禁止
raise BusinessError(code=ErrorCode.VALIDATION_ERROR, message=str(e))
```

### ❌ 错误：硬编码消息不用 data

```python
# 反模式 - 禁止
raise BusinessError(code=ErrorCode.VALIDATION_ERROR, message="银行账户不存在")
```

应该改为：

```python
raise BusinessError(
    code=ErrorCode.BANK_ACCOUNT_NOT_FOUND,
    data={"bank_account_id": bank_account_id}
)
```

### ❌ 错误：英文消息

```python
# 反模式 - 禁止
raise BusinessError(code=ErrorCode.VALIDATION_ERROR,
    message=f"category '{category}' not in {EXPENSE_CATEGORIES}")
```

## 会计错误处理

会计相关错误统一使用 `accounting_engine.AccountingError`（带 `accounting_rule` + `calculation_detail`），
注册在 `main.py:85`。

禁止使用 `models_finance.AccountingError`（该无 handler 类已废弃）。

```python
# ✅ 正确
from accounting_engine import AccountingError, AccountingErrorCode

raise AccountingError(
    code=AccountingErrorCode.INVOICE_AMOUNTS_NOT_BALANCED,
    message=f"发票金额不平衡：不含税 {a} + 税额 {b} ≠ {c}（差额 {diff}）",
    accounting_rule="《小企业会计准则》§二/2.1 发票金额计算",
    calculation_detail={...}
)
```

## 前端错误展示

前端必须通过 `handleError()` 统一处理，禁止直接读取 `err.response?.data?.detail`：

```javascript
// ✅ 正确
catch (e) { handleError(e, { defaultMsg: '保存失败' }) }

// ❌ 反模式 - 禁止
catch (e) { ElMessage.error(`失败: ${e.response?.data?.detail}`) }
```

## 反模式清单（Error-AP）

| # | 反模式 | 后果 | 修复方式 |
|---|--------|------|----------|
| Error-AP1 | `message=str(e)` | 暴露 Python 实现细节给用户 | 改用 `data={}` 或语义化消息 |
| Error-AP2 | 硬编码消息不用 data 模板 | 消息不一致，难以维护 | 走 ERROR_REGISTRY 模板 |
| Error-AP3 | 不同语义错误共用同一 ErrorCode（特别是 VALIDATION_ERROR） | 前端无法针对性处理 | 加专用 ErrorCode |
| Error-AP4 | 英文/中英混杂消息 | 用户看不懂 | 全中文消息 |
| Error-AP5 | 自建错误类（AccountingError 重复定义） | 异常落入 generic 500 handler | 统一用已有错误类 |
| Error-AP6 | 前端直接 ElMessage.error 读 detail | 丢失结构化信息 | 用 handleError() |
| Error-AP7 | INTERNAL_ERROR 用于域规则保护 | 语义错误，前端显示"联系管理员" | 用 409/403 对应 code |
