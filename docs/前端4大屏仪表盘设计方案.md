# 前端 4 大屏仪表盘设计方案

> 遵循 PR-2（报表直击痛点），纯前端实现，零后端改动

---

## 1. 目标

原 Dashboard 展示的是"今日采购/销售"等运营细节，老板打开首页最想知道的 4 个问题是：

| 老板问 | 对应大屏 |
|--------|----------|
| **这月赚了多少？** | 月度损益速览 |
| **仓库压了多少钱？** | 库存资金占用 |
| **客户欠我多少钱？** | 应收/应付汇总 |
| **这个月该交多少税？** | 税务速算 |

**要求**：首页一眼看到，排版舒服，数字大而清晰。

---

## 2. 数据映射（纯现有 API）

### 2.1 月度损益速览

| 指标 | API | 字段 |
|------|-----|------|
| 本月收入 | `GET /reports/profit?start_date=本月1日&end_date=今天` | `total_revenue` |
| 本月成本 | 同上 | `total_cost` |
| 本月毛利 | 同上 | `total_profit` |
| 本月费用 | `GET /expenses?start_date=本月1日&end_date=今天` | 前端聚合 `items[].amount` |
| **净利润** | 前端计算 | 毛利 - 费用 |
| 销售笔数 | `GET /reports/profit` | `sale_count` |

**边界情况**：
- 本月无销售 → 全部显示 ¥0.00，显示"暂无销售"
- 跨月切换 → 日期参数动态计算

### 2.2 库存资金占用

| 指标 | API | 字段 |
|------|-----|------|
| 库存总价值 | `GET /reports/overview` | `total_stock_value` |
| 库存总量 | 同上 | `total_inventory_quantity` |
| 商品种类 | 同上 | `total_products` |
| 预警商品数 | `GET /inventory/alerts` | 返回数组 `.length` |
| 负库存商品数 | `GET /reports/overview` | `negative_stock_count` |

### 2.3 应收/应付汇总

| 指标 | API | 字段 |
|------|-----|------|
| 应收总额 | `GET /sales?payment_status=unpaid&page_size=1000` | 前端聚合 `items[].total_price` |
| 未回款客户数 | 同上 | `total`（客户数） |
| 应付总额 | `GET /purchases?payment_status=unpaid&page_size=1000` | 前端聚合 `items[].total_price` |
| 未付款供应商数 | 同上 | `total` |

> ⚠️ **纯前端局限**：应收/应付通过加载未结清订单在前端计算，数据量大时需分页兜底。前端显示标注"基于未结清订单估算"。
> **远期优化**：新增 `GET /finance/receivable/summary` 后端聚合端点。

### 2.4 税务速算

| 指标 | API | 字段 |
|------|-----|------|
| 本月增值税 | `GET /tax-report/monthly?year=当前&month=当前` | `tax_payable` |
| 本季所得税 | `GET /income-tax-report?year=当前&quarter=当前` | `tax_amount` |
| 税费合计 | 前端计算 | 增值税 + 所得税 |
| 所属期 | 同上 | `period_start` ~ `period_end` |

**边界情况**：
- 本月无发票 → 增值税显示 ¥0.00
- 当前季度 != 申报季 → 显示"本季尚未申报"

### 3.2 配色方案

每张卡片一个主题色，4 色循环：

| 卡片 | 主题色 | 渐变色 | 图标 |
|------|--------|--------|------|
| 损益速览 | `#409EFF` 蓝 | `linear-gradient(135deg, #409EFF, #337ECC)` | `TrendCharts` |
| 库存资金 | `#67C23A` 绿 | `linear-gradient(135deg, #67C23A, #529B2E)` | `Box` |
| 应收应付 | `#E6A23C` 橙 | `linear-gradient(135deg, #E6A23C, #CF9236)` | `Coin` |
| 税务速算 | `#9B59B6` 紫 | `linear-gradient(135deg, #9B59B6, #7D3C98)` | `Document` |

### 3.3 设计细节

- **大数字**：主指标用 `font-size: 32px; font-weight: 700`，千分位展示
- **次要指标**：12px 灰色小字列表，hover 时显示 tooltip 说明
- **卡片**：圆角 12px，背景白色，左侧 4px 主题色竖条
- **hover 动效**：`transform: translateY(-4px); box-shadow` 微悬浮
- **加载态**：`v-loading` + skeleton 骨架屏
- **空态**：四卡各自独立空态，互不影响
- **响应式**：>=1400px 2×2，<1400px 1列堆叠

## 4. 组件结构

### 4.1 文件改动

| 文件 | 操作 | 说明 |
|------|------|------|
| `views/Dashboard.vue` | 重写 | 新布局容器 + 4 大屏 + 趋势图 + 预警表 |
| `api/finance.js` | 新增 3 个函数 | `getCurrentMonthProfit`, `getUnpaidSales`, `getUnpaidPurchases` |
| `components/` | 新增 1 个 | `BossCard.vue` 通用大屏卡片组件 |

### 4.2 不涉及后端改动

所有数据通过组合已有 API 调用在前端聚合：

```
Dashboard.vue
├── useDashboardData() (composable)
│   ├── loadProfitLoss() → financeApi.getProfitReport({start_date, end_date})
│   ├── loadInventory() → financeApi.getOverview() + productsApi.getAlerts()
│   ├── loadReceivable() → ordersApi.getSales({payment_status:'unpaid'}) + getPurchases(...)   ← 前端聚合
│   ├── loadTax() → invoicesApi.getTaxReportMonthly(year, month) + getIncomeTaxReport(year, quarter)
│   └── loadTrend() → financeApi.getTrend({days})   ← 复用现有
├── BossCard.vue (×4)
│   ├── slot: icon
│   ├── slot: main-value（大幅数字）
│   └── slot: details（明细列表）
├── ECharts 趋势图（复用现有）
└── 库存预警表（复用现有）
```

### 4.3 useDashboardData composable 接口

```js
// composables/useDashboardData.js
const {
  profitLoss,      // ref: { total_revenue, total_cost, gross_profit, total_expenses, net_profit, sale_count }
  inventory,       // ref: { total_stock_value, total_quantity, product_count, low_stock_count, negative_stock_count }
  receivable,      // ref: { total_receivable, unpaid_customer_count, total_payable, unpaid_supplier_count }
  tax,             // ref: { vat_payable, income_tax_payable, total_tax, period_label }
  loading,         // ref: boolean
  loadAll,         // () => Promise — 加载全部数据
  loadTrend,       // (days) => Promise — 加载趋势
  reload           // () => Promise — 刷新全部
} = useDashboardData()
```

## 6. 已知局限（纯前端方案）

| 问题 | 原因 | 后续优化 |
|------|------|----------|
| 应收/应付汇总不准 | 通过加载未结清订单前端聚合，不支持分页大数据 | 新增 `GET /finance/receivable/summary` |
| 月度损益不含费用 | `GET /reports/profit` 不含费用字段，需额外调 expenses API | 增强 profit 端点含 expense 汇总 |
| 税务速算需两趟请求 | 增值税 + 所得税分开调用 | 新增 `GET /tax-report/dashboard-summary` |
