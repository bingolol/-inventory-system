# 单一真相源原则 (Single Source of Truth)

> **目的**: 定义系统中每个财务/库存数据字段的唯一权威来源（真相源），供代码审核和审计使用。
> **读者**: 开发者、代码审核者、财务审计人员
> **版本**: v1.0 | 2026-06-28

---

## 一、原则定义

**单一真相源**（Single Source of Truth, SSOT）要求：每个业务数据只有一个权威写入入口（引擎层），所有读取方（报表层、导出层、凭证层）必须从该入口取数，禁止绕过引擎读取主数据的静态字段。

### 核心逻辑链

```
业务操作（采购/销售/收款/付款）
        ↓
   引擎层写入真相源        ← 唯一写入入口
   (StockMove / Inventory / LedgerAccountBalance)
        ↓
   报表层读取真相源        ← 所有读取方必须从这里取
   (利润表 / 资产负债表 / 税务报表 / 导出)
```

### 违反后果

| 反模式 | 后果 |
|--------|------|
| 报表读 `Product.purchase_price` 算成本 | 成本不反映实际采购价（移动加权平均），利润算错 → 所得税算错 |
| 报表读 `qty × purchase_price` 算库存价值 | 库存价值不等于总账 1405 余额 → 资产负债表不平 |
| 凭证兜底用 `purchase_price` | 总账 6401 成本错误 → 连带污染所有报表 |
| 硬编码 `0.13` 算销项税 | 无法支持 9%/6% 税率商品 → 增值税申报错误 |

## 二、真相源映射表

### 2.1 库存成本

| 数据需求 | 真相源（正确） | 禁止读取（错误） | 写入入口 |
|----------|---------------|-----------------|---------|
| 销售出库单位成本 | `SaleItem.unit_cost` | `Product.purchase_price` | `InventoryEngine.outbound()` 锁定 |
| 库存总价值 | `Inventory.total_value` | `qty × Product.purchase_price` | `InventoryEngine.inbound/outbound()` 更新 |
| 期间出库成本合计 | `Σ(StockMove.total_cost WHERE source_type='sale_order' AND quantity<0)` | `Σ(SaleItem.qty × Product.purchase_price)` | `InventoryEngine._record_outbound()` 写入 |
| 期间入库成本合计 | `Σ(StockMove.total_cost WHERE quantity>0)` | `Σ(PurchaseItem.qty × unit_price)` | `InventoryEngine.inbound()` 写入 |
| 移动加权平均成本 | `Inventory.average_cost` | `Product.purchase_price` | `InventoryEngine.inbound()` 重算 |

### 2.2 税务数据

| 数据需求 | 真相源（正确） | 禁止读取（错误） | 写入入口 |
|----------|---------------|-----------------|---------|
| 销项税额 | `Σ(Invoice.tax_amount WHERE direction=OUT)` 或 `output_tax` 参数 | `total_revenue × Decimal('0.13')` 硬编码 | 发票创建时写入 |
| 进项税额 | `Σ(Invoice.tax_amount WHERE direction=IN AND certified AND special)` | — | 发票认证时写入 |
| 增值税税率 | `Account.taxpayer_type` → 推导；行项 `item.tax_rate` | 硬编码 `Decimal('0.13')` | 账本创建 / 订单行项 |
| 城建税 | `VATResult.surcharge_urban_construction` | ~~`surcharge_stamp`~~（已重命名） | `AccountingEngine.calculate_vat()` 计算 |
| 企业所得税率 | `AccountingEngine.calculate_income_tax()` 返回 | 硬编码 | 引擎根据纳税人类型计算 |

### 2.3 总账数据

| 数据需求 | 真相源（正确） | 禁止读取（错误） | 写入入口 |
|----------|---------------|-----------------|---------|
| 期间收入 | `LedgerAccountBalance(6001)` 贷方发生额 | `Σ(SaleOrder.total_price)` | `FinanceEngine.record_sale()` 过账 |
| 期间成本 | `LedgerAccountBalance(6401)` 借方发生额 | `Σ(SaleItem.qty × purchase_price)` | `FinanceEngine.record_sale()` 过账 |
| 期间费用 | `LedgerAccountBalance(6601)` 借方发生额 | `Σ(Expense.amount)` | 费用创建时过账 |
| 应付账款 | `LedgerAccountBalance(2202)` 贷方余额 | `Σ(PurchaseOrder WHERE unpaid)` | `FinanceEngine.record_purchase()` 过账 |
| 应收账款 | `LedgerAccountBalance(1122)` 借方余额 | `Σ(SaleOrder WHERE unpaid)` | `FinanceEngine.record_sale()` 过账 |

## 三、引擎层职责边界

### 3.1 InventoryEngine（库存引擎）

**唯一职责**: 所有库存移动的写入入口

```
写入: StockMove（不可变真相源） + Inventory（缓存）
方法: inbound() / outbound() / reverse() / force_outbound()
规则: StockMove 一旦写入禁止修改/删除，错误修正必须通过 reverse() 红冲
```

**成本计算逻辑**（移动加权平均法，依据《小企业会计准则》§一/1.3）:
```
入库: avg_cost = (old_value + new_cost) / (old_qty + new_qty)
出库: unit_cost = inv.average_cost（锁定到 SaleItem.unit_cost）
```

### 3.2 FinanceEngine（财务引擎）

**唯一职责**: 所有会计凭证的生成入口

```
写入: AccountMove + AccountMoveLine + LedgerAccountBalance
方法: record_purchase() / record_sale() / reverse_purchase() / reverse_sale()
规则: 凭证借方合计 = 贷方合计（复式记账平衡）
```

### 3.3 AccountingEngine（会计准则引擎）

**唯一职责**: 税务计算（无状态纯计算）

```
方法: calculate_vat() / calculate_income_tax() / calculate_invoice_amounts()
规则: 不读写数据库，只做纯数学计算，输入→输出可复现
```

## 四、审核检查清单

### 4.1 代码审核检查项

审核任何涉及成本/税务/库存的代码时，逐项检查：

- [ ] **COGS 来源**: 成本计算是否读 `SaleItem.unit_cost`？是否禁止读 `Product.purchase_price`？
- [ ] **库存价值来源**: 库存价值是否读 `Inventory.total_value`？是否禁止读 `qty × purchase_price`？
- [ ] **税率来源**: 增值税税率是否从 `Account.taxpayer_type` 推导或行项 `item.tax_rate` 读取？是否禁止硬编码 `0.13`？
- [ ] **销项税来源**: 销项税是否从发票明细 `Invoice.tax_amount` 汇总？是否禁止用 `total_revenue × 硬编码税率` 估算？
- [ ] **纳税人类型来源**: 所得税计算是否从 `Account.taxpayer_type` 读取并正确映射？是否禁止硬编码 `'small_micro'`？
- [ ] **Decimal 精度**: `sum()` 返回值是否用 `_d()` 包裹（防空列表返回 `int 0`）？`.quantize()` 是否只对 `Decimal` 调用？
- [ ] **字段命名**: 城建税字段是否使用 `surcharge_urban_construction`？是否禁止使用 `surcharge_stamp`（印花税）？

### 4.2 数据审核检查项

- [ ] **BS 持平**: `total_assets == total_liabilities + total_equity`（误差 ≤ 0.01）
- [ ] **IS = BS 利润**: 利润表 `net_profit` == 资产负债表 `period_profit`
- [ ] **库存一致**: `Inventory.total_value` == `Σ(StockMove.total_cost WHERE quantity>0) - Σ(StockMove.total_cost WHERE quantity<0)`
- [ ] **COGS 一致**: 利润表 `cost_of_goods_sold` == `Σ(StockMove.total_cost WHERE source_type='sale_order')`

## 五、本次修复变更记录（2026-06-28）

### 5.1 问题 1: COGS 用 purchase_price 而非移动加权平均成本

**违反准则**: 《会计实务》§6.1 — 出库成本必须用 `SaleItem.unit_cost`（锁定时的 `average_cost`）

| 文件 | 修改内容 |
|------|---------|
| `crud/reports.py` | 利润报表成本: `product.purchase_price` → `item.unit_cost` |
| `routers/income_tax.py` | 经营口径成本: 同上 |
| `crud/finance.py` | 所得税预缴表成本: 同上 |
| `engine_journal.py` | 凭证兜底: `purchase_price` → 查 `StockMove.unit_cost`，缺失时报错 |

### 5.2 问题 2: 库存价值用 purchase_price 而非 total_value

**违反准则**: 信源审计 — 应读引擎维护的 `Inventory.total_value` 缓存

| 文件 | 修改内容 |
|------|---------|
| `crud/reports.py` | 库存价值: `qty × purchase_price` → `inv.total_value` |
| `routers/export.py` | 导出库存价值: 同上 |

### 5.3 问题 4: 所得税预缴表硬编码纳税人类型

**违反准则**: 应从 `Account.taxpayer_type` 读取

| 文件 | 修改内容 |
|------|---------|
| `crud/finance.py` | `taxpayer_type='small_micro'` → 读 `account.taxpayer_type` 并映射 |
| `routers/income_tax.py` | `account.taxpayer_type` 直接传入 → 映射 `small_scale` → `small_micro` |

**映射规则**: `small_scale`（VAT口径）→ `small_micro`（所得税口径），适用 5% 实际税负。

### 5.4 问题 5: 城建税字段命名为 surcharge_stamp（印花税）

**违反准则**: 7% 是城建税率（城市维护建设税），非印花税

| 文件 | 修改内容 |
|------|---------|
| `accounting_engine.py` | `VATResult.surcharge_stamp` → `surcharge_urban_construction` |
| `schemas/finance.py` | `VATDeclaration.surcharge_stamp` → `surcharge_urban_construction` |
| `crud/finance.py` | 输出字段名同步更新 |
| 4 个测试文件 | 断言字段名同步更新 |

### 5.5 问题 7: 增值税税率硬编码 0.13

**违反准则**: 应从发票明细汇总销项税，支持 13%/9%/6% 多税率商品

| 文件 | 修改内容 |
|------|---------|
| `accounting_engine.py` | `calculate_vat()` 加 `output_tax` 参数，优先用发票汇总值 |
| `crud/finance.py` | `generate_vat_declaration()` 传入 `output_tax` |
| `routers/tax.py` | `calculate_vat()` 调用传入 `output_tax` |
| `engine_finance.py` | `_vat_rate()` 加注释说明默认税率用途 |

### 5.6 问题 8: sum([]) 返回 int 0 陷阱

**违反准则**: 空数据集 `sum()` 返回 `int 0`，后续 `.quantize()` 崩溃

| 文件 | 修改内容 |
|------|---------|
| `crud/reports.py` | 6 处 `sum(...)` → `_d(sum(...))` |
| `crud/invoices.py` | 5 处 `sum(...)` → `_d(sum(...))` |

## 六、小微企业所得税声明

### 法规依据

《财政部 税务总局关于小微企业和个体工商户所得税优惠政策的公告》(2023年第12号)

### 计算规则

| 条件 | 税率 | 实际税负 | 法规依据 |
|------|------|---------|---------|
| 年应纳税所得额 ≤ 300万 | 减按25%计入 × 20%税率 | **5%** | 2023年第12号公告 |
| 年应纳税所得额 > 300万 | 法定税率25% | 25% | 企业所得税法 |

### 代码实现

```
AccountingEngine.calculate_income_tax():
  if taxpayer_type == 'small_micro' and profit <= 3,000,000:
      tax_payable = profit × 0.25 × 0.20  # = profit × 5%
      reduction_item = "小型微利企业减免（≤300万，实际税负5%）"
  else:
      tax_payable = profit × 0.25  # 法定税率
```

### 纳税人类型映射

系统中存在两种纳税人类型口径，需正确映射：

| 口径 | 字段值 | 用途 | 来源 |
|------|--------|------|------|
| VAT 口径 | `small_scale` / `general` | 增值税申报 | `Account.taxpayer_type` |
| 所得税口径 | `small_micro` / `general` | 企业所得税计算 | `AccountingEngine.calculate_income_tax()` |

**映射规则**: `small_scale` → `small_micro`（小规模纳税人通常同时也是小型微利企业）

## 七、第二批修复变更记录（2026-06-29）

### 7.1 问题 6: 小规模增值税普票/专票分开计算 + 免税门槛

**违反法规**: 《关于增值税小规模纳税人减免增值税政策的公告》(2026.1.1-2027.12.31)
- 季度销售额 ≤30万：**普票免征增值税**，专票不减兔（减按1%）
- 季度销售额 >30万：普票专票均减按1%征收

| 文件 | 修改内容 |
|------|---------|
| `accounting_engine.py` | `calculate_vat()` 加 `ordinary_revenue`/`special_revenue` 参数；小规模分支重写为三档逻辑 |
| `crud/finance.py` | `generate_vat_declaration()` 按发票类型拆分收入汇总 |
| `routers/tax.py` | 税务计算接口按发票类型拆分传入 |

### 7.2 亏损不抛异常，返回 0

**违反准则**: 《小企业会计准则》§5.5 — 亏损不缴税

| 文件 | 修改内容 |
|------|---------|
| `accounting_engine.py` | `calculate_income_tax()` 亏损时返回 `tax_payable=0`，不再抛 `INCOME_TAX_PROFIT_NEGATIVE` 异常 |

### 7.3 个体工商户不缴企业所得税

**违反法规**: 《个体工商户个人所得税计税办法》 — 个体户缴个人所得税，不缴企业所得税

| 文件 | 修改内容 |
|------|---------|
| `accounting_engine.py` | `calculate_income_tax()` 加 `entity_type` 参数；`personal` 时返回 0 |
| `crud/finance.py` | `generate_income_tax_prepayment()` 传 `entity_type=account.type` |
| `routers/income_tax.py` | 所得税报表接口传 `entity_type=account.type` |
