# 项目管理-项目详情页优化方案（体验+效率优先）

> 版本：v1.0 | 日期：2026-04-29 | 范围：前端为主（后端可选增强） | 状态：可执行交付

---

## 一、目标与非目标

### 1.1 目标（只优先两件事）

1. **体验**：打开项目详情 3 秒内看懂项目状态与关键数字（收入/成本/利润/风险）
2. **效率**：在详情内快速定位明细（筛选/搜索/汇总），并能追溯自动收入来源（销售单）

### 1.2 非目标（本轮不做）

- 不做复杂权限体系/多角色审批
- 不做大范围后端重构（除非为效率闭环必须）
- 不做全量分页重写（先做本地筛选+懒加载，数据量大再引入后端分页）

---

## 二、现状（基于当前实现）

- 前端项目详情位于 `frontend/src/views/Projects.vue` 的 `el-dialog + el-tabs`
- 详情数据来自 `GET /api/projects/{id}/cost-income`：返回 project + costs + incomes + purchases
- 常用操作（新增成本/编辑成本/编辑收入）通过多个弹窗完成，编辑后会调用 `viewManagedDetail({id})` 全量重拉
- 收入中 `source_type='sale_order'` 仅提示“自动”，未利用 `source_id` 做追溯

---

## 三、信息架构与交互（体验提升核心）

### 3.1 详情展示形态：Dialog → Drawer（推荐）

- 将项目详情从 `el-dialog` 改为 `el-drawer`（右侧抽屉）
- 抽屉支持：
  - 宽度 900~1100（默认 980）
  - “全屏/退出全屏”按钮（可选）
  - 关闭前确认（仅在有未保存编辑状态时）

#### 3.1.1 顶部 Sticky 概览区（必须有）

顶部固定区展示：
- 项目名 + 状态 Tag（ongoing/completed/cancelled）
- 客户名、开始日期
- KPI 卡片（见 3.2）
- 风险提示条（未收/未开票）
- 快捷动作：添加成本 / 添加收入（手动）/ 查看销售单（可选）

### 3.2 KPI 与风险（第一屏必须看到）

从 `managedDetail` 即可计算（不依赖后端改动）：

- **总收入**：`managedDetail.total_income`
- **总成本**：`managedDetail.total_cost`
- **利润**：`managedDetail.profit`（正绿负红）
- **待收款金额**：sum(income.amount - income.received_amount) for payment_status in (pending, partial)
- **已收金额**：sum(income.received_amount)
- **未开票金额**：sum(income.amount) where invoice_status != '已开'（或按你现有口径）

> 备注：若存在自动收入（sale_order）但无 `received_amount`，仍按“待收”计算逻辑展示风险。

## 四、Tabs 设计（效率提升核心）

### 4.1 Tab：成本明细（默认 Tab）

#### 4.1.1 筛选条（第一行）

- 成本类型：`cost_type`
- 支付方式：`payment_method`
- 发票状态：`invoice_status`
- 日期范围：`cost_date`（YYYY-MM-DD）
- 关键字：匹配 `product_name/supplier_name/notes`
- 动作区：`添加成本`（固定在右侧）

#### 4.1.2 表格列建议

- 类型、商品/数量（材料类一行可合并展示）、金额（排序+合计）、支付方式、发票状态、供应商、日期、备注、附件、操作

#### 4.1.3 编辑体验（必须提升）

当前 `costEdit` 只允许编辑 amount。建议升级为：
- 若 cost_type == 材料：允许编辑 `product_id/quantity/amount`（amount 仍为唯一真实值）
- 若非材料：编辑 `amount` + 其他辅助字段（notes/发票/供应商等可选）

> 若本轮不想扩大编辑范围，至少把“材料类不可编辑 product/quantity”写到 UI 提示中，避免误解。

### 4.2 Tab：收入明细

#### 4.2.1 筛选条

- 来源：手动/销售单自动（`source_type`）
- 收款状态：`payment_status`
- 发票状态：`invoice_status`
- 日期范围：`income_date`
- 关键字：notes
- 动作区：`添加收入（手动）`（可选）

#### 4.2.2 表格增强

- 表格上方显示“当前筛选下”汇总：待收/部分收取/已收 的金额合计
- 自动收入（sale_order）行展示：
  - 标签：销售单自动
  - 按钮：**查看销售单**（追溯来源）

#### 4.2.3 自动收入追溯（必须落地一种）

实现方式二选一：

- **方案A（最小改动，推荐先做）**：点击“查看销售单”调用 `GET /api/sales/{source_id}`，弹出一个只读抽屉/弹窗展示销售单明细（单号、客户、项目、items、金额、状态、日期、deduct_inventory）。
- **方案B（体验更好，需后端增强）**：项目详情接口直接返回 `sales` 列表；收入行直接显示单号并可定位。

### 4.3 Tab：关联采购单

增强点：
- 状态筛选、日期范围、本地关键字（供应商/单号）
- 增加“查看采购单”动作（同上：可弹窗只读或跳转 Purchases）

### 4.4 Tab：关联销售单（建议新增，闭环效率）

字段：单号、客户、金额、状态、日期、`deduct_inventory`

实现方式：
- 若后端支持 `projects/{id}/cost-income` 返回 `sales`：直接展示
- 否则需要新增 API（见第六章）

## 五、组件拆分（降低维护成本，避免 Projects.vue 继续膨胀）

建议拆分（先拆不改逻辑，再逐步增强）：

1. `frontend/src/components/projects/ProjectDetailDrawer.vue`
   - props: `projectId`, `visible`
   - emits: `close`, `updated`
   - 负责：拉取详情、顶部概览、tab 管理、懒加载触发

2. `frontend/src/components/projects/ProjectCostTab.vue`
   - props: `costs`, `projectId`
   - emits: `refresh`
   - 负责：筛选/搜索/汇总、表格、添加/编辑/删除成本

3. `frontend/src/components/projects/ProjectIncomeTab.vue`
   - props: `incomes`, `projectId`
   - emits: `refresh`, `openSaleOrder(sourceId)`
   - 负责：筛选/汇总、表格、编辑/删除手动收入、自动收入追溯入口

4. （可选）`ProjectOrdersTab.vue`
   - props: `purchases`, `sales`
   - 负责：关联单据 tab（采购/销售）

> 路径可按你现有项目结构调整，但建议放在 `components/projects/` 下，避免 views 过重。

## 六、后端接口（可选增强，但建议为闭环补齐）

### 6.1 现有接口（不变）

- `GET /api/projects/{id}/cost-income`

### 6.2 建议增强（v1：仅增加关联销售单）

在返回体中增加：

```json
{
  "sales": [
    {
      "id": 123,
      "order_no": "SO20260429-...",
      "customer_name": "客户",
      "total_price": 1000,
      "status": "completed",
      "sale_date": "2026-04-29",
      "deduct_inventory": false
    }
  ]
}
```

> 如果暂不增强后端，也可直接用 `GET /api/sales/{id}` 做追溯（方案A）。

## 七、实施步骤（建议拆 3 个增量）

### Step 1：结构升级（体验立竿见影）

- Dialog → Drawer
- 顶部 Sticky 概览区 + KPI/风险
- Tabs 懒加载（至少避免无意义重渲染）

### Step 2：效率增强（筛选/汇总/追溯）

- 成本/收入筛选条 + 本地过滤
- 当前筛选合计展示
- 自动收入“查看销售单”（方案A：直接调 `/api/sales/{id}`）

### Step 3：闭环补齐（可选）

- 项目详情接口返回关联 sales
- 详情页增加“关联销售单”Tab

---

## 八、验收标准（体验+效率）

1. **打开详情第一屏**能看到：状态/客户/日期 + 总收入/总成本/利润 + 待收/未开票风险
2. 成本明细可按至少 4 个维度筛选（类型/支付/发票/日期）+ 关键字，并显示筛选合计
3. 收入明细可按来源/收款/发票/日期筛选，并显示待收/已收汇总
4. 自动收入可追溯：点击“查看销售单”能看到对应销售单（至少只读）
5. 编辑/删除后不需要关闭重开即可看到更新（可局部刷新当前 tab）
