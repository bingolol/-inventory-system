## 🏷️ 测试分类

### 单元测试 (unit)

**目的**: 测试独立的函数、类、值对象

**特点**:
- 不依赖数据库
- 不依赖网络
- 执行速度快
- 测试业务规则

**标记**: `@pytest.mark.unit` (可选，当前未强制)

**示例**:
```python
# tests/unit/test_money.py
class TestMoneyArithmetic:
    def test_add(self):
        assert (Money("10.00") + Money("20.00")).amount == Decimal("30.00")
```

### 集成测试 (integration)

**目的**: 测试模块间交互，需要数据库

**特点**:
- 需要 SQLite 数据库
- 测试 Command → ORM → Database 链路
- 测试事务边界

**标记**: `@pytest.mark.integration` (可选)

**示例**:
```python
# tests/integration/test_cancel_sale_order.py
def test_cancel_sale_order_restores_inventory(db):
    # Given: 创建销售单并扣减库存
    # When: 取消销售单
    # Then: 库存恢复
```

### E2E 测试 (e2e)

**目的**: 端到端流程测试

**特点**:
- 使用 FastAPI TestClient
- 测试完整 HTTP 请求流程
- 模拟真实用户操作

## ✍️ 测试命名规范

### 文件命名

```
test_<被测试模块>.py
```

示例:
- `test_money.py` → 测试 `domain/money.py`
- `test_cancel_sale_order.py` → 测试取消销售单流程

### 类命名

```python
class Test<被测试功能>:
    """功能描述"""
```

示例:
```python
class TestMoneyConstruction:
    """Money 构造与基本属性"""
```

### 函数命名

```python
def test_<行为>_<条件>_<预期结果>():
```

示例:
```python
def test_should_return_error_when_inventory_insufficient():
    ...

def test_should_deduct_stock_when_sale_order_created():
    ...
```

## 📝 测试结构 (AAA 模式)

```python
def test_should_do_something():
    # Arrange (准备)
    input_data = ...
    expected = ...
    
    # Act (执行)
    result = function_under_test(input_data)
    
    # Assert (断言)
    assert result == expected
```

或使用中文注释:

```python
def test_should_do_something():
    # Given
    input_data = ...
    expected = ...
    
    # When
    result = function_under_test(input_data)
    
    # Then
    assert result == expected
```

## 🔧 Fixtures

### 全局 Fixtures

**文件**: `tests/conftest.py`

```python
"""测试全局配置 — 将 backend 目录加入 sys.path"""
import sys
import os

BACKEND_DIR = os.path.join(os.path.dirname(__file__), "..", "backend")
BACKEND_DIR = os.path.abspath(BACKEND_DIR)
if BACKEND_DIR not in sys.path:
    sys.path.insert(0, BACKEND_DIR)
```

### 集成测试 Fixtures

**文件**: `tests/integration/conftest.py`

```python
@pytest.fixture(scope="session", autouse=True)
def _init_db():
    """确保数据库初始化"""
    init_db()

@pytest.fixture
def db():
    """提供数据库 session，测试后自动关闭"""
    session = SessionLocal()
    try:
        yield session
    finally:
        session.close()
```

## ⚠️ 测试禁忌

### ❌ 不要这样做

```python
# 1. 不要测试实现细节
def test_internal_method():  # 不要测试私有方法
    ...

# 2. 不要依赖测试顺序
# 测试之间应该独立

# 3. 不要在测试中硬编码 ID
def test_get_product():
    product = db.query(Product).first()  # 不要假设 ID=1
    ...

# 4. 不要跳过断言
def test_something():
    result = do_something()
    # 忘记 assert = 测试永远通过

# 5. 不要在单元测试中使用真实数据库
def test_create_product():
    db = SessionLocal()  # 单元测试不应该依赖数据库
    ...
```

### ✅ 应该这样做

```python
# 1. 测试行为，不测试实现
def test_should_calculate_total():
    order = create_order(items=[...])
    assert order.total == expected_total

# 2. 每个测试独立
def test_case_a():
    ...
def test_case_b():
    ...

# 3. 使用查询而不是硬编码
def test_get_product():
    product = db.query(Product).filter(...).first()
    assert product is not None

# 4. 每个测试都有断言
def test_something():
    result = do_something()
    assert result == expected

# 5. 使用 fixture 管理依赖
def test_with_db(db):
    # db fixture 自动提供
    ...
```

## 🚀 运行测试

```bash
# 运行全部测试
pytest

# 运行特定目录
pytest tests/unit/
pytest tests/integration/

# 运行特定文件
pytest tests/unit/test_金额计算.py

# 运行特定测试函数
pytest tests/unit/test_金额计算.py::TestMoneyArithmetic::test_add

# 显示详细输出
pytest -v

# 显示 print 输出
pytest -s

# 只运行失败的测试
pytest --lf

# 失败后停止
pytest -x
```

## 🎯 Agent 测试选择指南

**根据任务类型选择测试**：

| 任务类型 | 应运行的测试 | 命令 |
|----------|--------------|------|
| 修改领域逻辑 | 单元测试 | `pytest tests/unit/` |
| 修改 Command/Handler | 单元 + 集成 | `pytest tests/unit/ tests/integration/` |
| 修改 API 路由 | 集成测试 | `pytest tests/integration/` |
| 修改前端 | E2E 测试 | `pytest tests/e2e/` |
| 不确定影响范围 | 全部测试 | `pytest` |
