## 📐 系统架构

### 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                      客户端层 (Client)                        │
│                   Vue 3 + Element Plus                       │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      API 网关层 (Gateway)                     │
│                    FastAPI + CORS + Auth                     │
└─────────────────────────────────────────────────────────────┘
                            │
            ┌───────────────┴───────────────┐
            ▼                               ▼
┌───────────────────┐           ┌───────────────────┐
│   路由层 (Router)  │           │   路由层 (Router)  │
│   读操作直接调用   │           │   写操作构建命令   │
└───────────────────┘           └───────────────────┘
            │                               │
            ▼                               ▼
┌───────────────────┐           ┌───────────────────┐
│   CRUD 层 (查询)  │           │ Command 层 (写操作)│
└───────────────────┘           └───────────────────┘
            │                               │
            │                               ▼
            │                   ┌───────────────────┐
            │                   │ Domain 层 (业务规则)│
            │                   └───────────────────┘
            │                               │
            └───────────────┬───────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      数据访问层 (Repository)                  │
│                   SQLAlchemy ORM + SQLite                    │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      事件总线 (EventBus)                      │
│                    日志记录 + 汇总重算                         │
└─────────────────────────────────────────────────────────────┘
```

### 核心机制

| 机制 | 文件 | 用途 |
|------|------|------|
| Command Pattern | `commands/base.py` | 封装所有写操作 |
| Unit of Work | `uow.py` | 事务边界管理 |
| EventBus | `events.py` | 事件发布/订阅 |
| BusinessError | `errors.py` | 统一业务错误处理 |
| AccountingEngine | `accounting_engine.py` | 会计准则统一计算引擎 + 输入/输出校验 |

### 关键表结构

#### Account (账本)
```sql
id              INTEGER PRIMARY KEY
name            VARCHAR(100) NOT NULL
type            VARCHAR(20)  -- 'company' | 'personal'
code            VARCHAR(50) UNIQUE
taxpayer_type   VARCHAR(20)  -- 'small_scale' | 'general'
created_at      DATETIME
```

#### Product (商品)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
name            VARCHAR(200) NOT NULL
sku             VARCHAR(50)
category        VARCHAR(100)
unit            VARCHAR(20)
purchase_price  DECIMAL(10,2)
sale_price      DECIMAL(10,2)
min_stock       INTEGER DEFAULT 0
track_inventory BOOLEAN DEFAULT 1
description     TEXT
created_at      DATETIME
updated_at      DATETIME
```

#### Inventory (库存)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
product_id      INTEGER REFERENCES Product(id)
quantity        DECIMAL(10,2) DEFAULT 0
UNIQUE(account_id, product_id)
```

#### PurchaseOrder (采购单)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
order_no        VARCHAR(50) UNIQUE
supplier_id     INTEGER REFERENCES Supplier(id)
order_type      VARCHAR(20) DEFAULT 'retail'
has_invoice     BOOLEAN DEFAULT 0
payment_method  VARCHAR(20) DEFAULT 'company'
payment_status  VARCHAR(20) DEFAULT 'unpaid'
status          VARCHAR(20) DEFAULT 'completed'
total_price     DECIMAL(12,2)
notes           TEXT
image_url       VARCHAR(500)
created_at      DATETIME
updated_at      DATETIME
```

#### SaleOrder (销售单)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
order_no        VARCHAR(50) UNIQUE
customer_id     INTEGER REFERENCES Customer(id)
deduct_inventory BOOLEAN DEFAULT 1
payment_status  VARCHAR(20) DEFAULT 'unpaid'
status          VARCHAR(20) DEFAULT 'completed'
total_price     DECIMAL(12,2)
notes           TEXT
image_url       VARCHAR(500)
created_at      DATETIME
updated_at      DATETIME
```

### 采购入库流程

```
用户创建采购单
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 1. 校验: items 非空, 无重复商品                       │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 2. 生成订单号: CG2026—6-19-14:30-01                 │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 3. 创建 PurchaseOrder 记录                          │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 4. 循环创建 PurchaseItem 记录                        │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 5. 检查 track_inventory                             │
│    True  → inv.quantity += item.quantity            │
│    False → 跳过                                     │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 6. 触发事件: purchase_order.created                 │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 7. 记录操作日志                                     │
└─────────────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────────────┐
│ 8. 事务提交 (unit_of_work)                          │
└─────────────────────────────────────────────────────┘
```

## 🔐 安全机制

### 多账本隔离

所有数据通过 `account_id` 隔离，API 请求必须携带 `X-Account-ID` 头。

```python
# 每个查询都必须过滤 account_id
items = db.query(Model).filter(Model.account_id == account_id).all()
```

### 操作审计

所有写操作记录到 `OperationLog` 表，包含：
- 操作类型 (create/update/delete/adjust)
- 实体类型和 ID
- 操作描述
- 操作员 (user/ai)

### 输入校验

- Pydantic Schema 校验请求格式
- BusinessError 校验业务规则
- SQLAlchemy 校验数据完整性

## 📊 性能考虑

### SQLite 优化

- 使用 WAL 模式提升并发读性能
- 合理使用索引
- 避免长时间锁定

### 查询优化

- 分页查询 (skip/limit)
- 按需加载字段
- 使用 joinedload 预加载关联
