### 核心机制

| 机制 | 文件 | 用途 |
|------|------|------|
| Command Pattern | `commands/base.py` | 封装所有写操作 |
| Unit of Work | `uow.py` | 事务边界管理 |
| EventBus | `events.py` | 事件发布/订阅 |
| BusinessError | `errors.py` | 统一业务错误处理 |
| AccountingEngine | `accounting_engine.py` | 会计准则统一计算引擎 + 输入/输出校验 |
| BankEngine | `engine_bank.py` | 银行引擎：BankAccount.balance 唯一更新入口 |
| FinanceEngine | `engine_finance.py` | 财务引擎：采购/销售会计凭证生成的唯一入口（封装 JournalEngine） |
| FixedAssetEngine | `engine_fixed_asset.py` | 固定资产引擎：折旧计提/处置的唯一入口 |
| InventoryEngine | `engine_inventory.py` | 库存引擎：StockMove 流水唯一入口，移动加权平均成本计算 |
| LedgerEngine | `engine_ledger.py` | 科目余额管理、试算平衡、非叶子保护 |
| JournalEngine | `engine_journal.py` | 5 种凭证模板（tax/operating） |
| ReceivableEngine | `engine_receivable.py` | 往来核销、4 桶账龄分析 |
| BankReconcileEngine | `engine_bank_reconcile.py` | 银行对账引擎：对账单匹配、差异分析 |
| TaxEngine | `engine_tax.py` | 税务引擎：增值税/所得税计算 + 申报表 |
| TaxCheckEngine | `engine_tax_check.py` | 税务自查引擎：进项/销项匹配校验 |
| FinanceIntegration | `finance_integration.py` | 业务入口调用 JournalEngine 的封装层，Command Handler 显式调用 |
| ReadonlyMiddleware | `middleware/readonly_middleware.py` | 只读中间件：阻止直接修改期初余额/发票/固定资产 |
| WritePermissionMiddleware | `middleware/write_permission.py` | 写权限中间件：基于角色的操作拦截 |
| AIGatewayMiddleware | `backend/ai_gateway.py` | AI 网关中间件：白名单校验 + AI 响应统一包装 |
| Auth | `routers/auth.py` | 登录认证：`POST /api/auth/login`，token 有效期 24h |
| AuthMiddleware | `account_dep.py:get_operator()` | 操作者识别：优先取 `Authorization: Bearer` token 用户名，次读 `X-Operator` |
| ORM 防护（before_update） | `models.py` / `models_finance.py` | `StockMove` / `FixedAssetDepreciation` / `AccountMove` 禁止 UPDATE |

### 关键表结构

#### Account (账本)
```sql
id              INTEGER PRIMARY KEY
name            VARCHAR(100) NOT NULL
type            VARCHAR(20)  -- 'company' | 'personal'
code            VARCHAR(50) UNIQUE
taxpayer_type   VARCHAR(20)  -- 'small_scale' | 'general'
created_at      DATETIME
```

#### Product (商品)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
name            VARCHAR(200) NOT NULL
sku             VARCHAR(50)
category        VARCHAR(100)
unit            VARCHAR(20)
purchase_price  DECIMAL(10,2)
sale_price      DECIMAL(10,2)
min_stock       INTEGER DEFAULT 0
track_inventory BOOLEAN DEFAULT 1
description     TEXT
created_at      DATETIME
updated_at      DATETIME
```

#### Inventory (库存缓存)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
product_id      INTEGER REFERENCES Product(id)
quantity        INTEGER DEFAULT 0       -- 当前库存（允许负数）
average_cost    DECIMAL(12,6) DEFAULT 0  -- 移动加权平均成本
total_value     DECIMAL(12,2) DEFAULT 0  -- 库存总金额
last_updated    DATETIME
UNIQUE(account_id, product_id)
```

#### StockMove (库存流水 — 库存真相源)
```sql
id              INTEGER PRIMARY KEY
product_id      INTEGER REFERENCES Product(id)
account_id      INTEGER REFERENCES Account(id)
quantity        DECIMAL(12,2)            -- 正数入库，负数出库
unit_cost       DECIMAL(12,6)            -- 当时单位成本
total_cost      DECIMAL(12,2)            -- 本次移动总金额
source_type     VARCHAR(50)              -- purchase_order / sale_order / adjustment
source_id       INTEGER
created_at      DATETIME
UNIQUE(source_type, source_id, product_id)
INDEX(product_id, account_id)
```

#### InventoryAdjustment (库存调整单)
```sql
id              INTEGER PRIMARY KEY
account_id      INTEGER REFERENCES Account(id)
product_id      INTEGER REFERENCES Product(id)
quantity        INTEGER                  -- 正=入库，负=出库
reason          VARCHAR(20)              -- inventory_count / spoilage / exchange / other
notes           TEXT

```

## 🚫 设计约束

### 库存创建路径（单一入口）

库存记录只能通过以下两种途径创建，禁止直接写入 `Inventory` 表：

| 途径 | 触发操作 | 库存方向 |
|------|----------|----------|
| **采购入库** | 创建采购单（`POST /api/purchases`） | 增加 |
| **期初余额** | 创建期初余额时录入库存（`POST /api/opening-balances`） | 增加 |

系统未提供独立的"新增库存"API。库存调整必须通过 `POST /api/inventory/adjustments` 创建调整单。

**依据**：采购入库有来源单据可追溯（采购单号+供应商），期初余额有初始化凭证可追溯。所有库存变动同时写入 `StockMove` 流水表（库存真相源），`Inventory` 表仅作为性能缓存。`StockMove` 一旦生成严禁修改删除，错误修正通过红冲调整单实现。

## 🔐 安全机制

### 多账本隔离

所有数据通过 `account_id` 隔离，API 请求必须携带 `X-Account-ID` 头。

```python
# 每个查询都必须过滤 account_id
items = db.query(Model).filter(Model.account_id == account_id).all()
```

### 操作审计

所有写操作记录到 `OperationLog` 表，包含：
- 操作类型 (create/update/delete/adjust)
- 实体类型和 ID
- 操作描述
- 操作员 (user/ai)

### 输入校验

- Pydantic Schema 校验请求格式
- BusinessError 校验业务规则
- SQLAlchemy 校验数据完整性
