## 新增功能速查

### 新增 API 端点

```
1. backend/enums.py        → 添加枚举（如需要）
2. backend/models.py       → 添加 ORM 模型（如需要）
3. backend/schemas/        → 添加 Pydantic Schema
4. backend/commands/       → 添加 Command + Handler
5. backend/crud/           → 添加查询函数
6. backend/routers/        → 添加路由端点
7. tests/                  → 添加测试
```

### 新增写操作 (Command 模式)

> 添加新的 CRUD 路由时优先使用 `utils/` 工具包的工厂函数：`get_or_404` / `op_create` / `op_update` / `op_delete` 等，见 **§🛠️ 常用工具函数**。

```python
# 1. 定义 Command
@dataclass
class MyCommand(Command):
    field: str = ""

# 2. 定义 Handler
@register(MyCommand)
class MyHandler(CommandHandler):
    def handle(self, cmd: MyCommand, db: Any) -> Any:
        # 业务逻辑
        _log(db, cmd.account_id, "create", "entity", entity.id, 
             "描述", operator=cmd.operator)
        db.flush()
        return entity

# 3. 在 commands/__init__.py 导出
from .my_commands import MyCommand
```

### 新增查询 (CRUD)

```python
# backend/crud/my_module.py
def list_items(db: Session, account_id: int, skip: int = 0, limit: int = 20):
    """查询列表，返回 (total, items)"""
    query = db.query(models.Item).filter(models.Item.account_id == account_id)
    total = query.count()
    items = query.offset(skip).limit(limit).all()
    return total, items
```

## 反模式红线 (Anti-Patterns)

### AP-1: 禁止在 CRUD 层写业务逻辑

❌ **错误**:
```python
# crud/products.py
def create_product(db, data):
    if data.quantity < 0:  # 业务校验不该在这里
        raise ValueError("数量不能为负")
    product = models.Product(**data)
    db.add(product)
    db.commit()  # CRUD 不该 commit
```

✅ **正确**:
```python
# commands/product_commands.py
@register(CreateProduct)
class CreateProductHandler(CommandHandler):
    def handle(self, cmd, db):
        # 业务校验在 Handler
        if cmd.quantity < 0:
            raise BusinessError(code=ErrorCode.VALIDATION_ERROR, ...)
        product = models.Product(...)
        db.add(product)
        db.flush()  # 只 flush，不 commit
```

---

### AP-2: 禁止在 Handler 外部调用 commit

❌ **错误**:
```python
# 任何地方
db.commit()  # 只有 uow.py 能 commit
```

✅ **正确**:
```python
# Router 中
with unit_of_work(db):
    result = dispatch(MyCommand(...), db)
# uow 自动 commit
```

---

### AP-3: 禁止假设数据存在

❌ **错误**:
```python
product = db.query(models.Product).first()  # 假设第一个就是目标
```

✅ **正确**:
```python
product = db.query(models.Product).filter(
    models.Product.account_id == account_id,
    models.Product.id == product_id,
).first()
if not product:
    raise BusinessError(code=ErrorCode.PRODUCT_NOT_FOUND, ...)
```

---

### AP-4: 禁止跳过 account_id 过滤

❌ **错误**:
```python
items = db.query(models.Item).all()  # 没有过滤账本
```

✅ **正确**:
```python
items = db.query(models.Item).filter(
    models.Item.account_id == account_id
).all()
```

### AP-5: 禁止在写操作中直接返回 ORM 对象

❌ **错误**:
```python
@router.post("")
def create_item(data: ItemCreate, db: Session = Depends(get_db)):
    item = dispatch(CreateItem(...), db)
    return item  # ORM 对象不可序列化
```

✅ **正确**:
```python
@router.post("")
def create_item(data: ItemCreate, db: Session = Depends(get_db)):
    with unit_of_work(db):
        item = dispatch(CreateItem(...), db)
    db.refresh(item)
    return schemas.ItemOut.model_validate(item)  # 返回 Schema
```

### AP-6: 禁止忽略 track_inventory 标志

❌ **错误**:
```python
# 所有商品都扣库存
inv.quantity -= quantity
```

✅ **正确**:
```python
# 检查是否跟踪库存
product = get_product(db, account_id, product_id)
if product.track_inventory:
    inv = get_or_create_inventory(db, account_id, product_id)
    inv.quantity -= quantity
```

### AP-7: 禁止硬编码金额计算

❌ **错误**:
```python
total = price * quantity  # 浮点数精度问题
```

✅ **正确**:
```python
from decimal import Decimal
from utils import Q2

total = (Decimal(str(price)) * Decimal(str(quantity))).quantize(Q2)
```

### AP-7.1: 禁止绕过 AccountingEngine 做会计计算

❌ **错误**:
```python
# 直接算增值税，绕过校验
tax_payable = output_tax - input_tax
if tax_payable < 0:
    tax_payable = Decimal('0')
```

✅ **正确**:
```python
from accounting_engine import AccountingEngine
_engine = AccountingEngine()

# 统一走引擎，自动校验输入 + 输出交叉验证
vat_result = _engine.calculate_vat(
    total_revenue=output_total,
    taxpayer_type=account.taxpayer_type,
    input_tax=input_tax
)
tax_payable = vat_result.tax_payable
```

**涉及的计算必须走 `accounting_engine.py`**：
- 发票金额（`calculate_invoice_amounts` / `validate_invoice_amounts`）
- 固定资产折旧（`calculate_depreciation_*`）
- 无形资产摊销（`calculate_intangible_amortization`）
- 增值税（`calculate_vat`）
- 企业所得税（`calculate_income_tax`）

### AP-9: 禁止跳过日志记录

❌ **错误**:
```python

    db.flush()
```

✅ **正确**:

### AP-12: 禁止在循环中创建数据库对象

❌ **错误**:
```python
for item in items:
    db.add(OrderItem(**item))
    db.flush()  # 每次都 flush
```

✅ **正确**:
```python
for item in items:
    db.add(OrderItem(**item))
db.flush()  # 批量 flush
```

### AP-13: 禁止忽略事件触发

❌ **错误**:
```python
def handle(self, cmd, db):
    order.status = OrderStatus.CANCELLED
    # 没有触发事件
    db.flush()
```

✅ **正确**:
```python
def handle(self, cmd, db):
    order.status = OrderStatus.CANCELLED
    emit("order.updated", db=db, order=order, operator=cmd.operator)
    db.flush()
```

### AP-14: 禁止为同一业务能力新增并行写端点变体

同一实体的增/改/删，对 AI 只能暴露 **1 个规范端点**。新能力（如"发票+固定资产联合入账"）
应作为现有规范端点的**可选嵌套字段**，而非新增 `/with-xxx` 并行端点。规范接口白名单见
`backend/ai_gateway.py` 的 `AI_CAPABILITIES`（唯一真相源，同时驱动 `GET /api/_ai/capabilities`）。

❌ **错误**:
```python
# 为"发票+固定资产"新增并行端点 → AI 多入口、舍入不一致、白名单膨胀
@router.post("/with-fixed-asset")
def create_invoice_with_asset(...): ...

@router.post("/with-fixed-asset-quick")  # 又一个变体
def create_invoice_quick_with_asset(...): ...
```

✅ **正确**:
```python
# 规范端点 /quick 增加可选嵌套字段，能力保留、入口统一
class InvoiceQuickCreate(BaseModel):
    ...  # 既有字段
    fixed_asset: Optional[FixedAssetBlock] = None  # 合并自原 /with-fixed-asset

@router.post("/quick")
def quick_create(invoice: InvoiceQuickCreate, ...):
    if invoice.fixed_asset is not None:
        # 复用已有 CreateInvoiceWithFixedAsset 命令，能力不丢
        result = dispatch(CreateInvoiceWithFixedAsset(...), db)
    else:
        dispatch(CreateInvoice(...), db)
```

> 删除已合并的变体端点前，须确认前端/AI 零调用（如本次 `/with-fixed-asset` 三个变体前端零引用）。
> 保留 Command 层逻辑（`CreateInvoiceWithFixedAsset` 等）——逻辑有价值，只是入口不再暴露。

### AP-15: 禁止脚本操作，必须通过 API

所有数据操作**禁止使用脚本**。调试/运维/批量操作必须通过 API 接口循环调用：
- 脚本绕过 API 会破坏事件总线、审计日志、库存引擎联动
- ORM 模型不在 API 层复用可能导致数据不一致
- 测试环境也不得直接写数据库，必须用 API

## 代码规范

### Python 风格

- 使用 `snake_case` 命名函数和变量
- 使用 `PascalCase` 命名类
- 使用类型提示
- 每个函数添加 docstring

### 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| Command | `动词 + 名词` | `CreateProduct`, `UpdateOrder` |
| Handler | `Command 名 + Handler` | `CreateProductHandler` |
| Schema | `名词 + In/Out` | `ProductCreate`, `ProductOut` |
| Router | 复数名词 | `products`, `orders` |

### 文件组织

- 每个模块一个文件
- 按功能分组
- 公共函数放 `base.py`

## 提交规范

### 提交范围

只提交 **源码 + 文档**，排除以下内容：

| 类型 | 排除原因 |
|------|----------|
| `node_modules/`、`dist/` | 构建产物 |
| 测试报告 `test-report.html`、`test-results/` | 生成文件 |
| 截图 `*.png`、`debug-*.png` | 调试产物 |
| 临时文件 `test_output.txt`、`test_results.txt`、`test_return.py` | 临时文件 |
| `.coverage`、`server.err` | 运行时生成 |
| 根目录测试报告 `*.md` | 一次性报表 |

### 提交粒度

- 后端功能改动、前端源码、文档更新，可合并在一个 commit 内
- 临时/调试文件不要 `git add .`，手动选择文件提交
