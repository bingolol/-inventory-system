### Python 风格

- 使用 `snake_case` 命名函数和变量
- 使用 `PascalCase` 命名类
- 使用类型提示
- 每个函数添加 docstring

### 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| Command | `动词 + 名词` | `CreateProduct`, `UpdateOrder` |
| Handler | `Command 名 + Handler` | `CreateProductHandler` |
| Schema | `名词 + In/Out` | `ProductCreate`, `ProductOut` |
| Router | 复数名词 | `products`, `orders` |

### 文件组织

- 每个模块一个文件
- 按功能分组
- 公共函数放 `base.py`

## 文档分类

所有 `docs/` 下的 `.md` 文件使用 `doc-type` 头部标记分类：

| 类型 | 说明 | 同步要求 |
|------|------|----------|
| `catalog` | 索引/清单类（INDEX.md, 文件索引.md, agent_skills.md） | 代码变更后必须更新 |
| `snapshot` | 调用链/数据因果链（代码调用逻辑图.md, 数据因果链.md, adr/设计决策.md） | 相关代码变更后必须重新验证 |
| `reference` | 参考手册/规范（开发速查表.md, 会计实务.md, 前端规范.md 等） | 无需随代码变更同步 |

**同步规则**: 代码变更后，执行 `grep -r "变更实体名" docs/` 定位需更新的文件。
