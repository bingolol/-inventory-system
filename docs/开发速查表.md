## 🏗️ 新增功能流程

### 后端开发顺序

1. **定义 ORM 模型** → `backend/models.py`
2. **定义 Pydantic Schema** → `backend/schemas/`
3. **实现 CRUD** → `backend/crud/`
4. **封装 Command（可选）** → `backend/commands/`
5. **添加 Router** → `backend/routers/`
6. **注册 Handler（可选）** → `backend/handlers.py`
7. **数据库迁移** → `backend/database.py` 补充 `ALTER TABLE`

### 前端开发顺序

1. **添加 API 接口** → `frontend/src/api/`
2. **创建视图页面** → `frontend/src/views/`
3. **添加路由（可选）** → `frontend/src/router/index.js`

### CRUD 写操作标准模板

```python
def create_xxx(db: Session, data: XxxCreate, account_id: int):
    try:
        obj = models.Xxx(**data.model_dump(), account_id=account_id)
        db.add(obj)
        db.flush()
        return obj
    except Exception:
        db.rollback()
        raise
```

### EventBus 优先级约定

| 优先级 | 用途 |
|--------|------|
| 10 | 写操作日志 |
| 90 | 项目汇总重算 |

> **v4 变更**：库存扣减/回补和收入联动已从 EventBus 移至 Command Handler 显式编排（`crud/inventory_ops.py` + `crud/income_ops.py`），EventBus 仅保留日志和汇总重算两项职责。

## 🐛 常见问题排查

### 问题 1：后端启动失败 - ModuleNotFoundError

| 排查步骤 | 检查项 |
|---------|--------|
| 1 | 虚拟环境是否已激活？ |
| 2 | `requirements.txt` 是否已安装？ |
| 3 | 命令是否在 `backend/` 目录下执行？ |

### 问题 2：API 返回 401 - 缺少 X-Account-ID

| 排查步骤 | 检查项 |
|---------|--------|
| 1 | 前端 `localStorage.currentAccountId` 是否存在？ |
| 2 | 测试请求头是否包含 `X-Account-ID: 1`？ |

### 问题 3：库存数据不一致

| 检查项 | 操作 |
|--------|------|
| 1 | 调用 `GET /api/projects/{id}/verify-invariants` 检查 |
| 2 | 调用 `POST /api/projects/{id}/reconcile` 强制重算 |
| 3 | 检查销售单 deduct_inventory 是否正确设置（项目专属和零售单必须true） |

### 问题 4：数据导出失败

| 检查项 | 说明 |
|--------|------|
| 1 | 必须使用 `api.exportFile()` 而不是 `window.open()` |
| 2 | 检查后端返回的 Content-Type |

### 问题 5：数据库锁定

| 解决方案 |
|---------|
| 确保只有一个 uvicorn 进程在运行 |
| 检查是否有 SQLite 客户端占用锁 |
| 停止所有进程，重启 uvicorn |

### 问题 6：422 校验错误

| 解决方案 |
|---------|
| 查看响应 detail 字段的错误描述 |
| 调用 `GET /api/enums` 获取合法枚举值 |
| 检查前端传递的枚举值是否正确 |

### 问题 7：金额字段显示报错 - toFixed is not a function

**现象**: 采购/销售/库存/费用页面控制台报错 `TypeError: xxx.toFixed is not a function`，表格数据无法正常显示。

**根本原因**: 后端API返回的金额字段是字符串类型（如 `"100.00"`），而前端模板直接使用 `.toFixed(2)` 方法，该方法仅在 Number 类型上可用。

**解决方案**:
- 使用 `formatMoney()` 工具函数统一处理金额格式化
- 导入方式: `import { formatMoney } from '../api/common'`
- 使用方法: `{{ formatMoney(row.total_price) }}` 替代 `{{ row.total_price?.toFixed(2) }}`

**涉及文件**:
- 工具函数: `frontend/src/api/common.js` 中的 `formatMoney()`
- 已修复页面: Purchases.vue, Sales.vue, Inventory.vue, Expenses.vue

### 问题 8：菜单点击路由不跳转

**现象**: 点击左侧菜单栏时，URL不变化或页面内容不更新，但直接访问URL正常。

**根本原因**: Element Plus 的 `el-menu` 内置 `router` 模式与 Vue Router 存在兼容性问题。

**解决方案**:
- 移除 `<el-menu>` 的 `router` 属性
- 添加 `@select="handleMenuSelect"` 事件处理器
- 手动调用 `router.push(index)` 进行路由跳转

**涉及文件**: `frontend/src/components/Layout.vue`

## 📐 三大不变量检查

### 不变量 I：库存一致性

```
inventory.quantity = Σ(完成的采购行数量) − Σ(已完成销售单行数量, deduct_inventory=true) − Σ(项目材料成本数量)
```

> 项目专属销售单和零售单(deduct_inventory=true)都会扣减库存，取消/删除时回补

**检查接口**: `GET /api/projects/{id}/verify-invariants`

### 不变量 II：项目收入一致性

`ProjectIncome` 记录与 `SaleOrder(project_id, status=completed)` 一一对应

**保障**: `UniqueConstraint(project_id, source_type, source_id)`

### 不变量 III：项目汇总一致性

`Project.total_cost` = 项目关联的已完成销售单中，每个商品的进价(purchase_price)×数量求和
`Project.total_income` = `ProjectIncome` 合计
`Project.profit` = total_income - total_cost
由 `update_project_summary()` 实时重算

**修复接口**: `POST /api/projects/{id}/reconcile`

## 🚫 反模式红线速查

> 详细规则见 `.joycode/rules/anti-patterns.md`，任何改动方案必须先通过自检。

### 三条元规则

| 元规则 | 机制 | 操作化 |
|--------|------|--------|
| MR-1 先读后写 | AI 从空白开始写，不扫描已有防护 | 写代码前找同类参照物，声明"参照了 xxx" |
| MR-2 捷径否决 | AI 优化"本次改动量最小" | 方案评审问"6个月后还成立吗？" |
| MR-3 全局卫生 | AI 只追加不清理 | 扫同类 + 删旧代码 + 验证无冲突 |

### 13条具体陷阱

| 编号 | 陷阱 | 一句话规则 | 自检问题 | 元规则 |
|------|------|-----------|---------|--------|
| AP-1 | 迁移清空数据 | 迁移必须幂等 | 有 DELETE/DROP 吗？幂等守卫？ | MR-2 |
| AP-2 | notes 字符串解析 | 关联用关联表 | 有在自由文本编码业务信息吗？ | MR-2 |
| AP-3 | EventBus 隐式联动 | 核心业务显式编排 | 有新增 EventBus handler 吗？ | MR-2 |
| AP-4 | toFixed 全局崩溃 | 类型边界系统防护 | 新字段有 formatMoney/Number 吗？修一处扫全站了吗？ | MR-1+MR-3 |
| AP-5 | 对账 API 404 | API 路径是契约 | 新路径已对照后端路由吗？ | MR-1 |
| AP-6 | `||` 导致 0 丢失 | 用 `??` 不用 `||` | 有 `|| null`/`|| 0` 做兜底吗？ | MR-1 |
| AP-7 | 重复 import/路由 | 先检查已有再添加 | 添加的代码是否已存在？ | MR-3 |
| AP-8 | 枚举硬编码 | 从单一来源读取 | 有硬编码枚举/路径/常量吗？ | MR-1 |
| AP-9 | 两套体系共存 | 旧代码被替代必须删除 | 旧实现已删除/废弃了吗？ | MR-2+MR-3 |
| AP-10 | inline 渐变样式 | 用设计系统 token | 有硬编码颜色/inline style 吗？ | MR-1+MR-2 |
