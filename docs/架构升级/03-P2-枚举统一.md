# P2：枚举单一真相源

> 阶段：P2 | 工作量：1天 | 风险：低 | 收益：消除枚举不同步bug
>
> **前置依赖**：无（可独立实施）
>
> **被依赖**：P5(Event handlers使用枚举类常量做条件)、P7(Pinia枚举store消费/api/enums)
>
> **独立实施说明**：即使P4/P5/P6/P7未实施，枚举统一仍然独立有效——后端硬编码替换为类常量可立即减少bug。

---

## ⚠️ 强制工作流程

```
1. 写方案    → 从架构视角出发，输出简洁优雅的代码方案（涉及哪些文件、改什么、怎么改）
2. 代码审查  → 语法检查 + 逻辑检查 + 架构检查，确保方案无遗漏无错误
3. 修正方案  → 根据审查结果修正方案
4. 再次审查  → 修正后的方案再次审查语法+逻辑+架构，确认无问题
5. 执行修改  → 按修正后的方案改代码
6. 测试验证  → 对更新过的功能和模块进行实际测试，确保100%落地无bug
7. 通过后更新文档进入下一个阶段
```

**禁止**：不写方案直接改代码、改完不测试就跳到下一个

---

## 目标

- 后端所有枚举集中在 `enums.py`，使用类常量
- `/api/enums` 端点暴露所有枚举（含状态枚举+中文标签）
- 前端从API获取枚举，不再硬编码

## /api/enums 端点扩展

```python
# main.py
from enums import ALL_ENUMS, ENUM_LABELS

@app.get("/api/enums")
def get_enums():
    """返回所有枚举值 + 中文标签"""
    return {
        "values": ALL_ENUMS,
        "labels": ENUM_LABELS,
    }
```
