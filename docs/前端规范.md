# 前端规范 v1.0

> 杂志风设计系统 + 代码规范 + 自检清单

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **杂志风** | 卡片16px圆角、浅阴影、充足留白、`#f5f6f8` 页面底色 |
| **深色侧栏** | `#141420→#1a1a2e` 渐变、白色文字、激活态高亮 |
| **主色调** | `--primary: #4f6ef7`（深蓝），非 Element Plus 默认蓝 |
| **状态色** | 成功 `#67c23a` / 警告 `#e6a23c` / 危险 `#f56c6c` / 信息 `#909399` |

## 2. 页面布局规范

### 2.1 标准页面结构

```
el-card
  ├── template #header
  │     └── .card-header
  │           ├── .page-title        ← 页面标题
  │           └── .card-header-actions ← 操作按钮组
  ├── .filter-bar                    ← 筛选栏
  │     ├── el-input/el-select/el-date-picker
  │     └── el-button type="primary" → 查询
  ├── el-table                       ← 数据表格
  │     └── el-table-column (详见 §3)
  └── .pagination-bar                ← 分页
```

**禁止**：
- ❌ 使用 `query-form` / `filter-form` / 自定义筛选栏类名 → 统一用 `.filter-bar`
- ❌ Header 中使用 inline `style="font-weight:600"` → 统一用 `.page-title`
- ❌ Header 中使用 inline flex 布局 → 统一用 `.card-header`
- ❌ `el-divider` 做段落分隔 → 改用 `.ob-section` 分区卡片

### 2.2 特殊页面（仪表盘 / 报表）

- **Dashboard**: Hero 深色渐变区 + MagazineCard 网格 + 图表
- **报表**: 顶部指标卡行 + 下方表格（不用 `el-statistic`）
- **期初余额**: `ob-section` 分区卡片 + `ob-section-tag` 彩色标签

## 3. 表格规范

### 3.1 列宽

| 列类型 | 规则 |
|--------|------|
| 数据列（名称、编码、金额、日期、备注） | `min-width="NNN"` |
| 操作列（编辑/删除按钮） | `width="NNN"` fixed |
| 展开/选择列 | `width="NNN"` keep |
| 状态列（已开票、支付状态） | `min-width="NNN"` + `align="center"` |

### 3.2 状态标签

```html
<!-- ✅ 统一使用 .status-badge -->
<span class="status-badge success">正常</span>
<span class="status-badge warning">偏低</span>
<span class="status-badge danger">负库存</span>
<span class="status-badge info">未开票</span>
<span class="status-badge primary">公司</span>
```

**禁止**：
- ❌ 表内用 `el-tag` 显示简单状态 → 改用 `.status-badge`
- ❌ `el-tag` 用于"是/否"、"有/无"列 → 改用 `.status-badge`

### 3.3 金额列

```html
<el-table-column label="金额" min-width="120" align="right">
  <template #default="{ row }">
    <span class="money">¥{{ formatMoney(row.amount) }}</span>
  </template>
</el-table-column>
```

## 5. CSS 规范

### 5.1 使用全局变量

```css
/* global.css 已定义的变量 */
--primary: #4f6ef7;
--primary-light: #eef1ff;
--primary-dark: #3b57d9;
--text-primary: #1d2129;
--text-secondary: #86909c;
--text-regular: #4e5969;
--bg-page: #f5f6f8;
```

### 5.2 作用域样式

- 全部使用 `<style scoped>`
- 禁止 inline 样式（弹窗 `width="500px"` 等 Element Plus 属性除外）
- 禁止全局样式泄漏到其他组件

### 5.3 暗色模式

`@media (prefers-color-scheme: dark)` 已定义在 `global.css`。新增页面无需额外处理。

## 6. 交互规范

| 场景 | 规范 |
|------|------|
| 数据加载 | `const loading = ref(false)` + `v-loading="loading"` |
| 错误处理 | `handleError(e, { defaultMsg: 'xxx失败' })` |
| 成功提示 | `ElMessage.success('xxx成功')` |
| 账本切换 | `useAccountAwareData(fetchFn)` |
| 按钮 loading | `:loading="saving"` 在保存按钮上 |
| 表格空状态 | `<el-empty description="暂无xxx" />` |

## 7. 反模式红线（前端）

### FP-1: 禁止硬编码宽度

❌ `<el-table-column prop="name" label="名称" width="120" />`
✅ `<el-table-column prop="name" label="名称" min-width="120" />`

### FP-2: 禁止 el-divider 做分区标题

❌ `<el-divider>流动资产类</el-divider>`
✅ 分区卡片 `.ob-section` + `.ob-section-tag`

### FP-3: 禁止 el-statistic 独立展示

❌ `<el-statistic title="营业收入" :value="x" />`
✅ 顶部指标卡 `.is-summary-item` / 或 MagazineCard

### FP-4: 禁止 inline style 替代 class

❌ `<span style="font-weight:600;color:var(--danger)">...</span>`
✅ `<span class="page-title">` / `<span class="status-badge danger">`

### FP-5: 禁止无 scoped <style> 的组件

❌ `<script>...</script>` 后直接结束
✅ 必须有 `<style scoped>`（可空）

### FP-6: 禁止用 type="primary" 做编辑按钮

❌ `<el-button type="primary" size="small">编辑</el-button>`
✅ `<el-button size="small" link type="primary">编辑</el-button>`

### FP-7: 分类列表页使用可筛选分类卡片

列表页（如费用管理）有分类属性时，顶部使用分类统计卡片：
```html
<div class="ex-stats">
  <div class="ex-stat" @click="filter = ''" :class="{ active: !filter }">全部</div>
  <div v-for="cat in categories" :key="cat.name" class="ex-stat"
       :style="{ borderLeftColor: cat.color }"
       @click="filterByCategory(cat.name)">
    <span>{{ cat.name }}</span>
    <span>{{ formatMoney(cat.total) }}</span>
  </div>
</div>
```
- 每张卡片有彩色左侧边框
- 点击切换筛选，选中态高亮
- 分类颜色映射在 `CATEGORY_COLORS` 常量中管理

### FP-8: 表单弹窗用分组卡片

表单字段超过 5 个时使用彩色分组卡片：
```html
<div class="form-group" style="border-left-color:#4f6ef7;">
  <div class="form-group-header">
    <span class="form-group-tag">组名</span>
  </div>
  <div class="form-group-body">
    <div class="form-field">
      <span class="form-label">字段名</span>
      <el-input v-model="..." />
    </div>
  </div>
</div>
```
- 每张卡片 4px 彩色左侧边框 + 12px 圆角
- 组名用药丸标签 `.form-group-tag`
- 字段用水平布局：label + input 同行

### FP-9: 财务页面必须展示计算公式

所有涉及计算结果的财务页面，必须展示计算过程和公式，让用户理解数字来源：

✅ **利润表**：展示每步计算公式
```
毛利润 = 营业收入 − 营业成本
¥128,600 − ¥85,200 = ¥43,400
```

✅ **财务总览健康指标**：展示公式
```
资产负债率 = 总负债 ÷ 总资产 × 100%
¥400,000 ÷ ¥1,000,000 × 100% = 40%
```

✅ **试算平衡表**：展示借贷合计是否平衡

✅ **对账**：展示期初+本期-已收/付=期末公式

❌ 禁止只显示数字不给计算过程

## 8. 自检清单

每个页面/组件完成后检查：

- [ ] 使用 `.page-title` + `.card-header` + `.filter-bar` 标准结构
- [ ] 表格列使用 `min-width`（操作/展开/选择列除外）
- [ ] 状态使用 `.status-badge` 而非 `el-tag`
- [ ] 金额使用 `.money` + `formatMoney()`
- [ ] 无 inline 样式
- [ ] 有 `<style scoped>`
- [ ] 使用 `handleError` 错误处理
- [ ] 使用 `useAccountAwareData` 自动加载
- [ ] 编辑按钮用 `link` 类型
- [ ] 无 `el-divider` 分区（除非是内容分隔线）
- [ ] 无 `el-statistic` 独立展示
