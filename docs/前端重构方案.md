# 前端代码重构优化方案

> 2026-06-21 | 基于 TDD refactoring 原则

## Context

前端代码存在多处重复：Suppliers.vue 与 Customers.vue 几乎完全相同（99行 vs 99行），Purchases.vue 与 Sales.vue 高度相似（getSummaries/loadOptions/handleSave 重复），`formatMoney` 被错误放置在 API 层且被 19 个文件引用，错误处理存在三种不一致模式，`api/common.js` 混合了 6 类不相关职责。

---

## Task 1: 统一 `formatMoney` 并迁移至工具层
**优先级**: 高 | **风险**: 低  
**文件**: `utils/format.js`, `api/common.js`, 19 个引用文件

- 在 `utils/format.js` 添加带千分位的 `formatMoney`（采用 `api/common.js` 的 `toLocaleString` 实现）
- 从 `api/common.js` 删除 `formatMoney` 定义，改为从 `utils/format.js` 重导出（保持兼容）
- 更新所有 import 路径

## Task 2: 合并 Suppliers.vue 和 Customers.vue 为通用 PartnerList 组件
**优先级**: 高 | **风险**: 低  
**文件**: `components/PartnerList.vue`(新建), `views/Suppliers.vue`, `views/Customers.vue`

- 新建 `components/PartnerList.vue`，通过 props 接收差异配置
- Suppliers.vue 和 Customers.vue 缩减为 ~10 行薄包装组件

## Task 3: 消除 `useOrderForm.js` 中新建/编辑表单的内部重复
**优先级**: 高 | **风险**: 中  
**文件**: `composables/useOrderForm.js`

- 提取内部工厂函数 `createItemActions(formRef)`，消除约 30 行重复代码
- 对外接口保持不变

## Task 4: 提取订单列表通用 `getSummaries` 方法至 composable
**优先级**: 中 | **风险**: 低  
**文件**: `composables/useOrderList.js`, `views/Purchases.vue`, `views/Sales.vue`

## Task 5: 统一错误处理策略
**优先级**: 中 | **风险**: 中  
**文件**: 所有视图文件

## Task 6: 拆分 `api/common.js` 按职责拆分
**优先级**: 中 | **风险**: 低  
**文件**: `api/common.js`, `api/accounts.js`(新建), `api/export.js`(新建), `api/upload.js`(新建), `api/backup.js`(新建)

## Task 7: Layout.vue 菜单数据驱动化 + 账本管理提取
**优先级**: 中 | **风险**: 低  
**文件**: `components/Layout.vue`, `composables/useMenuConfig.js`(新建), `composables/useAccountManagement.js`(新建)

## Task 8: Expenses.vue 清理
**优先级**: 中 | **风险**: 低

## Task 9: Personal.vue 使用 enumsStore 替代直接 API 调用
**优先级**: 低 | **风险**: 低

## Task 10: `usePagination.js` 命名优化
**优先级**: 低 | **风险**: 低

## Task 11: global.css 瘦身
**优先级**: 低 | **风险**: 中

---

## 执行顺序

| 阶段 | Tasks | 目标 |
|------|-------|------|
| **第一阶段：基础清理** | 1, 8, 9, 10 | 统一工具函数、清理局部问题 |
| **第二阶段：核心去重** | 2, 3, 4 | 消除最大重复代码 |
| **第三阶段：架构优化** | 5, 6, 7 | 统一错误处理、拆分职责、数据驱动 |
| **第四阶段：收尾** | 11 | CSS 瘦身（所有页面稳定后） |

## 验证方式

每个 Task 完成后：`cd frontend && npm run build` 确认无编译错误
