## 1. 商品管理

**用途**: 管理所有商品/服务的基础信息，是采购、销售、库存业务的基础数据。

**核心功能**:
- 商品CRUD（创建/查询/修改/删除）
- 商品分类管理
- SKU编码管理
- 可追踪库存标记（`track_inventory`）

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/products` | 商品列表（分页/搜索/SKU/分类筛选） |
| POST | `/api/products` | 创建商品 |
| GET | `/api/products/categories/list` | 获取所有分类 |
| GET | `/api/products/{id}` | 商品详情 |
| PUT | `/api/products/{id}` | 修改商品 |
| DELETE | `/api/products/{id}` | 删除商品（有业务记录时不可删） |

**业务规则**:
- `track_inventory=true` 的商品参与库存管理（实物商品）
- `track_inventory=false` 的商品跳过所有库存操作（如人工服务）
- 有采购或销售记录的商品不可删除
- 删除商品时同步删除对应库存记录

**Command**: `CreateProduct` / `UpdateProduct` / `DeleteProduct`

## 2. 库存管理

**用途**: 实时查看和调整商品库存数量，是库存不变量（不变量I）的核心验证对象。

**核心功能**:
- 库存列表查询（分页/搜索/分类/低库存预警）
- 库存盘点（手动调整数量）
- 低库存预警

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/inventory` | 库存列表（仅显示 track_inventory=true 的商品） |
| GET | `/api/inventory/alerts` | 低库存预警列表 |
| PUT | `/api/inventory/{product_id}` | 库存盘点调整 |

**业务规则**:
- 库存由采购入库、销售出库、项目领料自动维护，一般不需要手动调整
- 库存盘点用于校准实际数量，与系统数量不一致时使用
- 库存不变量I：`库存 = 采购入库 - 销售出库(deduct_inventory=true) - 项目材料领料`

**Command**: `AdjustInventory`

## 3. 采购管理

**用途**: 管理采购订单，记录从供应商采购商品的信息，并自动入库。

**核心功能**:
- 采购单CRUD
- 采购入库（自动增加库存）
- 采购单状态变更（待定/已完成/已取消）
- 关联项目与供应商
- 关联发票

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/purchases` | 采购单列表（分页/日期/状态/关键词/类型筛选） |
| POST | `/api/purchases` | 创建采购单（默认COMPLETED状态，自动入库） |
| GET | `/api/purchases/{id}` | 采购单详情 |
| PUT | `/api/purchases/{id}` | 修改采购单（含items/状态变更） |
| DELETE | `/api/purchases/{id}` | 删除采购单 |

**业务规则**:
- 创建采购单默认 `status=COMPLETED`，立即入库
- 取消采购单时扣减库存（仅 `track_inventory=true` 的商品）
- 恢复采购单时重新入库
- 同一采购单内同一商品不可重复添加
- 人工采购单（`order_type=PURCHASE_LABOR`）由项目成本驱动，禁止手动删除/修改items
- 修改items时：旧行回补→删旧行→建新行→按新状态扣减

**Command**: `CreatePurchaseOrder` / `CancelPurchaseOrder` / `DeletePurchaseOrder` / `UpdatePurchaseOrderItems` / `UpdatePurchaseOrderFields`

## 4. 销售管理

**用途**: 管理销售订单，记录向客户销售商品的信息，并自动扣减库存和生成项目收入。

**核心功能**:
- 销售单CRUD
- 销售出库（自动扣减库存，`deduct_inventory=true` 时）
- 项目收入自动生成
- 销售单状态变更
- 零售单与项目专属销售单区分

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/sales` | 销售单列表（分页/日期/状态/关键词/类型筛选） |
| POST | `/api/sales` | 创建销售单 |
| GET | `/api/sales/{id}` | 销售单详情 |
| PUT | `/api/sales/{id}` | 修改销售单（含items/状态变更） |
| DELETE | `/api/sales/{id}` | 删除销售单 |

**业务规则**:
- **零售单**（`order_type=RETAIL`）：`deduct_inventory` 必须为 `true`，禁止关联 `project_id`
- **项目专属销售单**（`order_type=PROJECT`）：`deduct_inventory` 必须为 `true`，由项目成本/完结联动生成
- 取消销售单时回补库存（仅 `deduct_inventory=true` 的订单）
- 恢复销售单时重新扣减库存
- 项目专属销售单禁止删除、禁止修改items、禁止切换项目
- 同一销售单内同一商品不可重复添加
- 项目关联的销售单完成时自动生成 `ProjectIncome`

**Command**: `CreateSaleOrder` / `CancelSaleOrder` / `RestoreSaleOrder` / `DeleteSaleOrder` / `UpdateSaleOrderItems` / `UpdateSaleOrderFields` / `CompleteProject` / `LinkProject`

## 5. 项目管理

**用途**: 管理项目的全生命周期，包括项目信息、成本归集、收入确认和利润计算。

**核心功能**:
- 项目CRUD
- 项目详情（成本+收入+汇总）
- 三大不变量实时验证
- 项目汇总手动重算
- 项目完结

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/projects` | 项目列表（分页/状态筛选） |
| GET | `/api/projects/list` | 项目简要列表（下拉选择用） |
| POST | `/api/projects` | 创建项目 |
| GET | `/api/projects/{id}/cost-income` | 项目详情（成本+收入+汇总） |
| PUT | `/api/projects/manage/{id}` | 修改项目 |
| DELETE | `/api/projects/manage/{id}` | 删除项目 |
| POST | `/api/projects/verify-invariants` | 三大不变量验证 |
| POST | `/api/projects/reconcile` | 强制重算项目汇总 |

**业务规则**:
- 项目汇总（`total_income`/`total_cost`/`profit`）由 `update_project_summary()` 实时重算
- `total_cost` = 项目关联的已完成销售单中每个商品的进价×数量求和
- 项目完结时自动生成项目型销售单和收入
- 有关联数据的项目不可删除

## 6. 项目成本管理

**用途**: 记录项目发生的各类成本（材料/人工/差旅等），并联动销售单和采购单。

**核心功能**:
- 项目成本CRUD
- 成本类型区分（材料/人工/差旅/其他）
- 材料成本联动：自动写入项目销售单 + 扣减库存
- 人工成本联动：自动写入人工采购单
- 成本-行项精确关联（`CostItemLink`）

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/costs` | 创建项目成本 |
| GET | `/api/costs` | 成本列表（按项目/类型筛选） |
| PUT | `/api/costs/{id}` | 修改成本 |
| DELETE | `/api/costs/{id}` | 删除成本（联动回补库存/删除行项） |

**业务规则**:
- 材料类成本：查找/创建项目专属销售单 → 写入SaleItem → 扣减库存
- 人工类成本：查找/创建人工采购单 → 写入PurchaseItem
- 删除成本时：回补库存/删除行项，同步更新项目汇总
- 每条成本通过 `CostItemLink` 精确关联到对应的 SaleItem/PurchaseItem

**Command**: `CreateProjectCost` / `DeleteProjectCost`

## 7. 项目收入管理

**用途**: 记录项目产生的收入，与销售单联动，是项目收入不变量（不变量II）的核心验证对象。

**核心功能**:
- 项目收入CRUD
- 收入来源追踪（`source_type`/`source_id`）
- 销售单自动生成收入
- 手动收入冲突检查

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/costs/incomes` | 创建项目收入 |
| GET | `/api/costs/incomes` | 收入列表 |
| PUT | `/api/costs/incomes/{id}` | 修改收入 |
| DELETE | `/api/costs/incomes/{id}` | 删除收入 |

**业务规则**:
- 销售单完成时自动创建 `ProjectIncome`（`source_type="sale_order"`）
- 手动创建收入时检查冲突：如果项目已有 sale_order 来源的收入，阻止手动创建
- `UniqueConstraint(project_id, source_type, source_id)` 防止重复收入记录
- 删除收入时同步更新项目汇总

**Command**: `CreateProjectIncome` / `UpdateProjectIncome` / `DeleteProjectIncome`

## 13. 期初余额

**用途**: 设置财务期初余额，为资产负债表提供起始数据。

**核心功能**:
- 期初余额CRUD
- 会计恒等式校验（资产 = 负债 + 权益）
- 日期唯一性校验

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/opening-balances` | 创建期初余额 |
| GET | `/api/opening-balances` | 期初余额列表 |
| GET | `/api/opening-balances/latest` | 最新期初余额 |
| GET | `/api/opening-balances/{id}` | 期初余额详情 |
| PUT | `/api/opening-balances/{id}` | 修改期初余额 |
| DELETE | `/api/opening-balances/{id}` | 删除期初余额 |

**业务规则**:
- 创建/更新时强制校验会计恒等式：资产 = 负债 + 所有者权益
- 同一日期的期初余额唯一
- 期初余额为资产负债表提供起始点

**Command**: `CreateOpeningBalance` / `UpdateOpeningBalance` / `DeleteOpeningBalance`

- 企业所得税报表

| 方法 | 路径 | 说明 |
|------|------|------|

## 20. 备份与导出

**用途**: 数据备份和导出，保障数据安全。

**核心功能**:
- 数据库热备份
- Excel/CSV导出
- 批量商品导入

**API端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/backup/hot` | 手动触发数据库热备份 |
| GET | `/api/export/inventory` | 导出库存 |
| GET | `/api/export/purchases` | 导出采购单 |
| GET | `/api/export/sales` | 导出销售单 |
| GET | `/api/export/report` | 导出报表 |
| GET | `/api/export/products` | 批量导出商品模板 |

**业务规则**:
- 热备份使用SQLite内置备份API，不影响在线使用
- 导出格式支持Excel（.xlsx）和CSV
- 批量商品导入使用Excel模板
