## 方案 1:发票表为单一真相源

### 目标
删除 3 张表(销售单/采购单/费用)的 `has_invoice` 布尔字段,改为从发票表派生查询。让发票表的关联记录成为"是否有发票"的唯一真相。

### 决策清单(grilling 已确认)

| # | 决策 | 选定 |
|---|------|------|
| 1 | 迁移策略 | A 一步到位(开发环境直接开干) |
| 2 | InvoiceLinkage 放哪层 | B' 放 `crud/invoice_linkage.py` 作领域服务(handler 调用,跟随 `inventory_ops.py` 先例) |
| 3 | link() 何时调用 | B 简化:关联就是 ORM 字段本身,不新增 link 调用;`validate_link_target` 只做防孤儿校验 |
| 4 | related_order_type 枚举约束 | A 优化:DB CheckConstraint 兜底 + schema pattern 友好报错 + validate_link_target 防孤儿(三层防御) |
| 5 | 删列迁移方式 | 沿用 `_migrate_numeric_fields` 重建表模式 + 补行数校验 |
| 6 | 5 处读取点改造 | A:schema 去字段,3 个构造函数填派生值 |

| 8 | validate 校验时机 | CreateInvoice handler 内,设 ORM 字段前校验目标存在 |

### 迁移脚本要点(`_migrate_drop_has_invoice`)

```
1. 备份整库 .pre_drop_has_invoice
2. 检测 sale_orders 是否已无 has_invoice 列 → 已迁移跳过
3. 对 3 张表(sale_orders/purchase_orders/expenses)逐个:
   a. RENAME 旧表 → __old_has_invoice
   b. create_all 建新表(无该列)
   c. INSERT 共有列数据
   d. 行数校验:旧表 count == 新表 count,不一致抛错(旧表未删,安全)
   e. DROP 旧表
4. PRAGMA foreign_keys 开关
```

## 方案 5:增值税走负债,缴费税走 Payment.tax 类型

### 目标
增值税属负债(应交税费 2221),不走费用。给 Payment 加 `tax` 付款类型,缴税时清偿负债,不碰利润表。附加税和所得税走费用(补类目)。

### 法规依据(docs/小企业会计准则.md)
- §六销售分录:销项税计入应交税费(2221),不进收入
- §六采购分录:进项税抵减负债,不进成本
- §三/3.1利润表公式:营业利润里无增值税
- §五科目映射:2221 应交税费 = 流动负债

### 决策清单

| # | 决策 | 选定 |
|---|------|------|
| 11 | payment_type 加 tax | enums/Schema/路由三处加枚举值 |
| 12 | EXPENSE_CATEGORIES 补类目 | 加"税金及附加"/"所得税";functional 加"税金及附加" |

### 改动文件清单

| 文件 | 改动 |
|------|------|
| `enums.py` | EXPENSE_CATEGORIES 加 `"税金及附加"`、`"所得税"`;EXPENSE_FUNCTIONAL_CATEGORIES 加 `"税金及附加"` |
| `schemas/payment.py` | payment_type 描述加 tax |
| `models.py` | Payment.payment_type 注释加 tax(已有列,无需迁移) |
| `routers/payments.py` | payment_type=tax 时关联应交税费(清负债),不碰利润表 |
| `routers/receipts.py` | 同上(收款侧,如退税) |

### 税种归属速查

| 税种 | 归属 | 走费用? | Payment.type | Expense.category |
|------|------|---------|-------------|-----------------|
| 增值税 | 负债(2221) | 否 | tax | — |
| 附加税 | 费用 | 是 | expense | 税金及附加 |
| 所得税 | 费用 | 是 | expense | 所得税 |

### 测试策略

1. **单元测试**:枚举值校验(税金及附加/所得税合法)
2. **集成测试**:
   - Payment type=tax → 银行存款减、应交税费清零、利润表无影响
   - Expense category=税金及附加 → 进费用、利润表体现
   - Expense category=所得税 → 进费用、利润表体现

## 全局约定(已写入 CONTEXT.md)

- **BR-1** 单一真相源:发票表关联是唯一真相
- **BR-2** 口径规则:经营口径取订单金额(含税),税务口径取发票金额(不含税)
- **BR-3** 开票同步:has_invoice=True 即已开票事实,无时间差
- **BR-4** 无票不查:小规模无票收入不强制计提销项,故意设计
- **BR-5** 税费分类:增值税走负债,所得税/附加税走费用
