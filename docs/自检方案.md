# 后端逻辑链·时间链·财税链自检方案

> 目标：每次代码改动后，按此方案逐层自检，确保 3 条链完整自洽。
> 方式：新建隔离账本跑脚本，不污染生产数据。

---

## 一、逻辑链自检（Logic Chain）

### 1.1 正向全流程（10步）

```
建账 → 期初 → 采购 → 付款 → 销售 → 收款 → 费用(公司付) → 费用(个人垫付) → 报销 → 工资计提 → 工资发放 → 固定资产 → 折旧
```

每步检查：
- [ ] API 返回 200，entity_id 非空
- [ ] 对应业务表有记录
- [ ] 有 `AccountMove` + `AccountMoveLine`（journal entry）
- [ ] 有 `StockMove`（库存操作时）
- [ ] 余额表 `LedgerAccountBalance` 借贷平衡

### 1.2 逆向全流程（4步）

```
取消采购(已付款) → BS验算
取消销售(已收款) → BS验算
删除采购 → BS验算  
删除销售 → BS验算
```

每步检查：
- [ ] 库存已回滚（StockMove 有 reversal 记录、Inventory 数量恢复）
- [ ] 凭证已冲红（AccountMove.is_reversal=True，借贷互换）
- [ ] 收/付款已冲销（有反向 Payment/Receipt 记录）
- [ ] BS 取消前后相等（取消 = 没发生过）

### 1.3 状态机校验

| 实体 | 合法转换 | 非法转换 |
|------|---------|---------|
| PurchaseOrder | pending → completed, completed → cancelled, completed → deleted | pending → deleted（有库存变动时） |
| SaleOrder | pending → completed, completed → cancelled, cancelled → completed(恢复) | 已删除 → 任何 |
| Invoice | 无状态机，只标记 is_reversed | — |

## 二、时间链自检（Time Chain）

### 2.1 业务日期 vs 系统日期

| 表 | 业务日期字段 | 取值来源 | 检查 |
|----|------------|---------|------|
| `StockMove.move_date` | ✅ 新增 | `_get_move_date()` 从源单据取 | 不能等于 `datetime.now()` |
| `AccountMove.date` | 现有 | `source.get("date")` | 等于业务日期，非系统时间 |
| `Payment.payment_date` | 现有 | API 传入 | 等于业务日期 |
| `Receipt.receipt_date` | 现有 | API 传入 | 等于业务日期 |

**测试命令：**
```python
# 检查 StockMove 不含当前时间
bad = db.query(StockMove).filter(
    func.date(StockMove.move_date) == func.date(datetime.now())
).count()
assert bad == 0, f"有 {bad} 条 StockMove 日期等于当前时间"
```

### 2.2 跨期隔离

**场景：** M1 创建数据，M2 创建数据，M1 的 BS 不应包含 M2 数据。

| 维度 | 隔离方式 | 验证方法 |
|------|---------|---------|
| 库存 | StockMove.move_date ≤ query_end | M1 BS inventory = 仅 M1 操作的库存 |
| AR/AP | SaleOrder/PurchaseOrder.sale_date/purchase_date ≤ query_end | M1 BS 不包含 M2 的应收应付 |
| 费用 | Expense.expense_date ≤ query_end | M1 BS expense = 仅 M1 的费用 |
| 利润 | 同口径过滤 | M1 IS 不含 M2 的收入/成本 |

**测试：**
```python
# M1 BS inventory + M2 BS inventory ≠ 总库存（确保不重复）
# M1期未 + M2期未 ≠ 总累加（M2期未已含M1）
assert abs(M1_BS.inventory + M2_period_only.inventory - M2_BS.inventory) < 0.01
```

### 2.3 时间边界

| 边界 | 正确写法 | 错误写法 |
|------|---------|---------|
| 日期比较 | `datetime` 比较 `datetime` | `date` 比较 `datetime` |
| 期末截止 | `query_end = date + 1day - 1sec (23:59:59)` | `date 00:00:00` |
| 期初起始 | `opening_date = date 00:00:00` | `opening_date = date` |
| Date 列比较 | `date_col <= query_end.date()` | `date_col <= query_end`（TypeError） |

### 2.4 折旧时点

小企业会计准则第三十一条：**当月增加下月提，当月减少下月停**

```python
months = (query_date.year - asset.start_date.year) * 12 + (query_date.month - asset.start_date.month)
if 0 < months <= asset.useful_life:  # months=0 表示当月，不提
    depreciation = ...
```

## 三、财税链自检（Finance-Tax Chain）

### 3.1 借贷平衡

全量检查 `AccountMoveLine`：

```sql
SELECT move_id, SUM(debit), SUM(credit)
FROM account_move_lines
GROUP BY move_id
HAVING ABS(SUM(debit) - SUM(credit)) > 0.01
```

期望：返回 0 行。

### 3.2 试算平衡表

```
总借方余额 = 总贷方余额
资产类余额(借) + 费用类余额(借) = 负债类余额(贷) + 权益类余额(贷) + 收入类余额(贷)
```

### 3.3 增值税验算

一般纳税人：
```
销项税额 = sum(SaleItem.total_price / 1.13 × 0.13)
进项税额 = sum(Invoice.tax_amount WHERE direction=IN, certified, special)
应纳增值税 = max(销项税额 - 进项税额, 0)
```

验算：
```python
assert abs(bs.tax_payable - max(output_tax - input_tax, 0)) < 0.01
```

### 3.4 利润表公式（小企业会计准则5.2）

```
校验1：营业毛利 = 营业收入 - 营业成本
校验2：营业利润 = 营业毛利 - 销售费用 - 管理费用 - 财务费用 - 税金及附加
校验3：利润总额 = 营业利润 + 营业外收入 - 营业外支出
校验4：净利润 = 利润总额 - 所得税费用
```
