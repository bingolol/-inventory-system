# 开发规范

> 开发规范速查 + 反模式红线 + 精简原则

---

## 精简原则（Pruning Principles）

| # | 原则 | 判据 |
|---|------|------|
| **P1** | 零调用即删除 | 函数/类/变量在整个 backend 中无 import 引用 → 删除 |
| **P2** | 三处同模即提取 | 同一段逻辑（5 行以上）在 3 个以上文件中复制 → 提取公共函数 |
| **P3** | 透传即合并 | Router 端点仅做 `dispatch` + `return`、无任何分支/循环/转换 → 用通用 dispatcher 替代 |
| **P4** | 一次性迁移不沉底 | 数据修复脚本放 `scripts/`，不在 `database.py` 中持久化 |
| **P5** | 字段映射不手写 | ORM→Schema 用 `model_validate` + `@computed_field`，禁止 `id=obj.id, name=obj.name` 式手工搬运 |

---

## 铁律

**动码前先问**：每次修改代码前（新建/编辑/删除文件），必须先向用户呈现改动计划并等待批准。

## 新增功能速查

### 新增 API 端点

```
1. backend/enums.py        → 添加枚举（如需要）
2. backend/models.py       → 添加 ORM 模型（如需要）
3. backend/schemas/        → 添加 Pydantic Schema
4. backend/commands/       → 添加 Command + Handler
5. backend/crud/           → 添加查询函数
6. backend/routers/        → 添加路由端点
7. tests/                  → 添加测试
```

### 新增写操作（Command 模式）

```python
# 1. 定义 Command
@dataclass
class MyCommand(Command):
    field: str = ""

# 2. 定义 Handler
@register(MyCommand)
class MyHandler(CommandHandler):
    def handle(self, cmd: MyCommand, db: Any) -> Any:
        # 业务逻辑
        _log(db, cmd.account_id, "create", "entity", entity.id,
             "描述", operator=cmd.operator)
        db.flush()
        return entity

# 3. 在 commands/__init__.py 导出
from .my_commands import MyCommand
```

### 新增查询（CRUD）

```python
# backend/crud/my_module.py
def list_items(db: Session, account_id: int, skip: int = 0, limit: int = 20):
    """查询列表，返回 (total, items)"""
    query = db.query(models.Item).filter(models.Item.account_id == account_id)
    total = query.count()
    items = query.offset(skip).limit(limit).all()
    return total, items
```

### 新增前端页面

```
1. frontend/src/views/MyPage.vue     → 页面组件
2. frontend/src/api/myApi.js         → API 请求
3. frontend/src/router/index.js      → 添加路由
```

## 反模式红线（Anti-Patterns）

### AP-1: 禁止在 CRUD 层写业务逻辑

❌ **错误**:
```python
# crud/products.py
def create_product(db, data):
    if data.quantity < 0:  # 业务校验不该在这里
        raise ValueError("数量不能为负")
    product = models.Product(**data)
    db.add(product)
    db.commit()  # CRUD 不该 commit
```

✅ **正确**:
```python
# commands/product_commands.py
@register(CreateProduct)
class CreateProductHandler(CommandHandler):
    def handle(self, cmd, db):
        if cmd.quantity < 0:
            raise BusinessError(code=ErrorCode.VALIDATION_ERROR, ...)
        product = models.Product(...)
        db.add(product)
        db.flush()
```

---

### AP-2: 禁止在 Handler 外部调用 commit

❌ **错误**: 任何地方 `db.commit()`

✅ **正确**: Router 中用 `with unit_of_work(db): dispatch(...)` 包裹

---

### AP-3: 禁止假设数据存在

❌ **错误**: `product = db.query(models.Product).first()`

✅ **正确**: 同时 filter `id` + `account_id`，查不到抛 `BusinessError`

---

### AP-4: 禁止跳过 account_id 过滤

❌ **错误**: `items = db.query(models.Item).all()`

✅ **正确**: `items = db.query(models.Item).filter(models.Item.account_id == account_id).all()`

---

### AP-5: 禁止在写操作中直接返回 ORM 对象

❌ **错误**: `return item`（ORM 对象不可序列化）

✅ **正确**: `return schemas.ItemOut.model_validate(item)`

---

### AP-6: 禁止忽略 track_inventory 标志

❌ **错误**: 所有商品都扣库存

✅ **正确**: 检查 `product.track_inventory` 再决定是否调 `InventoryEngine`

---

### AP-7: 禁止硬编码金额计算

❌ **错误**: `total = price * quantity`（浮点数精度问题）

✅ **正确**: `total = (Decimal(str(price)) * Decimal(str(quantity))).quantize(Q2)`

---

### AP-7.1: 禁止绕过 AccountingEngine 做会计计算

涉及的计算必须走 `accounting_engine.py`：发票金额、折旧、摊销、增值税、企业所得税。

---

### AP-8: 禁止在 Handler 中直接抛 HTTP 异常

❌ **错误**: `raise HTTPException(status_code=404)`

✅ **正确**: `raise BusinessError(code=ErrorCode.ORDER_NOT_FOUND, ...)`

---

### AP-9: 禁止跳过日志记录

写操作必须在 `db.flush()` 后调用 `_log()`。

---

### AP-10: 禁止在测试中使用真实数据库

✅ **正确**: `sqlite:///:memory:` 测试数据库

---

### AP-11: 禁止忽略错误码

❌ **错误**: `raise ValueError("商品不存在")`

✅ **正确**: `raise BusinessError(code=ErrorCode.PRODUCT_NOT_FOUND, data={"product_id": product_id})`

---

### AP-12: 禁止在循环中逐条 flush

❌ **错误**: `for item in items: db.add(...); db.flush()`

✅ **正确**: `for item in items: db.add(...)` → 循环后一次性 `db.flush()`

---

### AP-13: 禁止忽略事件触发

状态变更后必须 `emit("order.updated", ...)`

---

### AP-14: 禁止为同一业务能力新增并行写端点变体

同一实体的增/改/删，对 AI 只暴露 1 个规范端点。新能力作为现有端点的可选嵌套字段。

---

### AP-15: 禁止脚本操作，必须通过 API

所有数据操作必须通过 API，禁止脚本直连数据库。违反 BR-17。

---

### AP-16: 修改前先 grep 下游消费者

修改 `engine_*.py`、`crud/*.py`、`finance_integration.py` 前必须追踪下游引用。

---

### AP-17: 委托 Task Agent 必须加禁项清单

Task Agent 会自主注入无关代码，委托时必须显式写明禁止项。

---

### AP-18: 破坏性 Git 操作必先 dry-run

`git clean`、`git reset --hard` 执行前必须 `-n` 预览。

---

### AP-19: 禁止手工 ORM→Schema 字段搬运

❌ **错误**:
```python
def _to_out(order):
    return schemas.OrderOut(
        id=order.id, order_no=order.order_no,       # ← 每个字段手写
        total_price=order.total_price_l1,
        created_at=order.created_at, ...
    )
```

✅ **正确**:
```python
# 1) Schema 定义中同名字段自动映射
class OrderOut(BaseModel):
    id: int
    order_no: str
    total_price: Decimal = Field(alias="total_price_l1")
    created_at: datetime
    # 2) 派生字段用 computed_field
    @computed_field
    @property
    def customer_name(self) -> str:
        return ""

# Router 中一行搞定
return OrderOut.model_validate(order)
```

**例外**：当 Schema 需要跨关联表取派生字段（如 `item.product.name`）且 Pydantic 不支持 ORM 懒加载时，可在 router 层手工组装，但必须提取为独立的 `_build_xxx_out()` 函数，禁止在 endpoint 内联写。

---

### AP-20: 禁止跨文件复制编排逻辑

库存+凭证的编排模式（`InventoryEngine.outbound → item.set_calculated_cost → FinanceEngine.record_sale`）在多个文件中重复出现。

❌ **错误**: `_lifecycle.py`、`_sale.py`、`_purchase.py` 各自实现同一段 20 行编排

✅ **正确**: 提取为 `OrderLifecycle` 上的一个方法，参数化 `force`、`direction`（in/out），所有调用方共用一个入口

---

### AP-21: 禁止各层各自查询 Account

Account 对象的纳税人类型、政策配置读取散布在 3 层 34 处。

❌ **错误**: 在任何需要 `taxpayer_type` 的地方重复 `db.query(Account).filter(...).first()`

✅ **正确**: 
```python
from policy.entity_profile import build_profile
profile = build_profile(db, account_id)  # 一次加载，含 taxpayer_type / income_type / surcharge_halved
```

---

### AP-22: 禁止 database.py 存放一次性迁移

❌ **错误**: 某次 bug 修复的数据迁移脚本（如 `_migrate_asset_code_*`、`_sync_bank_account_*`）永久沉在 `database.py` 的 `init_db()` 中

✅ **正确**: 一次性迁移脚本放 `scripts/`，执行完即删除。`database.py` 只保留通用结构迁移（`_auto_migrate_columns`、`_create_immutable_triggers`）

---

### AP-23: 禁止 Schema `__init__.py` 全量重导出

❌ **错误**: `__all__` 列 50+ 类名，含重复项

✅ **正确**: 各模块按需 `from schemas.order import OrderCreate`，`__init__.py` 只导出被 5+ 文件引用的公共类型。

---

### AP-24: 禁止路由器端点纯透传

❌ **错误**:
```python
@router.post("")
def create_xxx(data: XxxCreate, db=Depends(get_db)):
    with unit_of_work(db):
        return dispatch(CreateXxx(...), db)
```
这种 5 行代码不做任何判断/转换/校验，仅仅是 HTTP→Command 适配器。

✅ **正确**: 合并到通用 CRUD router，或用统一的 `dispatch_endpoint(model, command_cls, schema_cls)` 工厂函数。

### Python 风格

- 使用 `snake_case` 命名函数和变量
- 使用 `PascalCase` 命名类
- 使用类型提示
- 每个函数添加 docstring

### 命名约定

| 类型 | 约定 | 示例 |
|------|------|------|
| Command | `动词 + 名词` | `CreateProduct`, `UpdateOrder` |
| Handler | `Command 名 + Handler` | `CreateProductHandler` |
| Schema | `名词 + In/Out` | `ProductCreate`, `ProductOut` |
| Router | 复数名词 | `products`, `orders` |

### 文件组织

- 每个模块一个文件
- 按功能分组
- 公共函数放 `base.py`

## 提交规范

### 提交范围

只提交 **源码 + 文档**，排除以下内容：

| 类型 | 排除原因 |
|------|----------|
| `node_modules/`、`dist/` | 构建产物 |
| 测试报告 `test-report.html`、`test-results/` | 生成文件 |
| 截图 `*.png`、`debug-*.png` | 调试产物 |
| 临时文件 | 临时文件 |
| `.coverage`、`server.err` | 运行时生成 |

### 提交粒度

- 后端功能改动、前端源码、文档更新，可合并在一个 commit 内
- 临时/调试文件不要 `git add .`，手动选择文件提交

## 常用工具函数

### `backend/utils/` 工厂函数

| 函数 | 用途 |
|------|------|
| `_d(val)` | 安全转换为 Decimal（None→0） |
| `parse_date(s)` / `parse_date_to_date(s)` | 解析 'YYYY-MM-DD' |
| `fmt_date(d)` / `fmt_datetime(dt)` | 格式化为 ISO 字符串 |
| `get_quarter_date_range(year, quarter)` | 季度起止日期（半开区间） |

### `backend/crud/base.py` 公共函数

| 函数 | 用途 |
|------|------|
| `_generate_order_no(db, prefix, business_date=None)` | 生成订单号（CG/XS 前缀 + 日期 + 序号） |
| `_log(db, account_id, operation, entity_type, entity_id, detail="", operator="user")` | 写操作日志（只 add+flush，不 commit） |
| `get_or_create_inventory(db, account_id, product_id)` | 获取或创建库存记录 |

### `backend/uow.py` 事务边界

| 函数 | 用途 |
|------|------|
| `unit_of_work(db)` | 写操作上下文管理器 |
| `read_only(db)` | 只读上下文 |

**规则**:
1. CRUD 函数只做 `db.flush()`，不调 `commit()` / `rollback()`
2. `_log()` 也只做 `flush()`，日志与业务数据在同一事务中
3. `HTTPException` 在 uow 内抛出也会触发 rollback
4. `db.refresh()` 需要在 uow 块外执行（commit 后才能 refresh）

### 事件总线 `backend/events.py`

| 函数 | 用途 |
|------|------|
| `on(event, priority=0, name=None)` | 事件注册装饰器 |
| `emit(event, **kwargs)` | 触发事件 |
| `emit_async(event, **kwargs)` | 异步触发事件 |

事件处理器注册在 `backend/handlers.py`（模块加载时自动注册）。
设计原则：**Emit-as-Log**（单一 seam）——日志写入只在 handler 中执行一次，Command 不得直接 `_log`。

### 命令分发 `backend/commands/base.py`

| 函数 | 用途 |
|------|------|
| `@register(cmd_type)` | Handler 注册装饰器 |
| `dispatch(cmd, db)` | 分发命令到对应 Handler |
