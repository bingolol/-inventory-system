## 三、修复内容详解

### 3.1 COGS 信源修正（问题 1）

**核心改动**: 所有成本计算从 `Product.purchase_price` 改为 `SaleItem.unit_cost`（移动加权平均出库成本）。

| 文件 | 改动 |
|------|------|
| `crud/reports.py` | 利润报表成本改用 `item.unit_cost` |
| `routers/income_tax.py` | 经营口径成本改用 `item.unit_cost` |
| `crud/finance.py` | 所得税预缴表成本改用 `item.unit_cost` |
| `engine_journal.py` | 凭证兜底从查 `StockMove.unit_cost` 获取，禁止用 `purchase_price` |
| `routers/export.py` | 导出库存价值改用 `Inventory.total_value` |

### 3.2 小规模增值税普票/专票分开计算（问题 6）

**法规依据**: 2026.1.1-2027.12.31 小规模纳税人减免增值税政策

**修复逻辑**:
```
季度总销售额 ≤30万：
  普票 → 免征增值税（tax = 0）
  专票 → 减按1%征收（tax = special_revenue × 1%）

季度总销售额 >30万：
  普票 → 减按1%征收
  专票 → 减按1%征收

附加税 → 基于实际缴纳的增值税（免税部分不计附加税）
```

**关键设计**: 免税门槛看季度总销售额（普票+专票合计），但只有普票享受免税，专票始终按1%缴税。

### 3.3 亏损不抛异常（问题 6a）

**法规依据**: 《小企业会计准则》§5.5 — 亏损不缴税

**修复**: `calculate_income_tax()` 亏损时返回 `tax_payable=0` + `reduction_item="亏损，不计提所得税"`，不再抛异常。与 `engine_tax.py` 月结的 `max(cumulative_profit * rate, 0)` 逻辑一致。

### 3.4 个体工商户不缴企业所得税（问题 6b）

**法规依据**: 《个体工商户个人所得税计税办法》

**修复**: `calculate_income_tax()` 增加 `entity_type` 参数：
- `company` → 正常计算企业所得税（5%/25%）
- `personal` → 返回 0（个体户缴个人所得税，系统不处理个税）

利用现有 `Account.type` 字段（`company/personal`），无需新增数据库字段。

### 3.5 其他修复

- **城建税字段重命名**（问题5）: `surcharge_stamp` → `surcharge_urban_construction`
- **增值税税率硬编码**（问题7）: `calculate_vat()` 加 `output_tax` 参数，从发票明细汇总
- **纳税人类型硬编码**（问题4）: 读 `Account.taxpayer_type` 并映射
- **Decimal 精度**（问题8）: `sum(...)` 统一用 `_d(sum(...))` 包裹

## 六、小微企业所得税声明

### 法规依据

《财政部 税务总局关于小微企业和个体工商户所得税优惠政策的公告》(2023年第12号)，执行至2027年12月31日。

### 计算规则

| 条件 | 税率 | 实际税负 |
|------|------|---------|
| 年应纳税所得额 ≤300万 | 减按25%计入 × 20%税率 | **5%** |
| 年应纳税所得额 >300万 | 法定税率25% | 25% |

### 主体类型规则

| 主体类型 | 字段值 | 缴纳税种 |

| 公司/有限责任公司 | `Account.type = "company"` | 企业所得税 |
| 个体工商户 | `Account.type = "personal"` | 个人所得税（系统不处理） |

## 八、审计方法论总结

本次审计使用"信源审计"方法论，核心原则：

1. **引擎层是唯一写入入口**: StockMove / Inventory / LedgerAccountBalance
2. **报表层必须从引擎取数**: 禁止读主数据静态字段（`Product.purchase_price`）
3. **税务计算必须从发票取数**: 禁止硬编码税率估算
4. **纳税人类型/主体类型从账本读取**: 禁止硬编码

所有修改均遵循"改一处、说明一处"原则，不改动未确认的代码。
