# 生产环境部署指南

## 一、首次安装（生产电脑）

### 前置条件

生产电脑需要安装以下软件（只需安装一次）：

| 软件 | 版本要求 | 下载地址 |
|------|---------|---------|
| Python | 3.10+ | https://www.python.org/downloads/ |
| Node.js | 18+ | https://nodejs.org/ |
| Git | 任意 | https://git-scm.com/download/win |

> **安装 Python 时**：务必勾选 `Add Python to PATH`
> **安装 Node.js 时**：选择 LTS 版本即可

### 安装步骤

1. 在生产电脑桌面新建一个文件夹，将 `一键安装.bat` 拷贝进去
2. 双击运行 `一键安装.bat`
3. 脚本会自动完成：
   - 检查环境（Python / Node.js / Git）
   - 从 GitHub 克隆项目代码到桌面
   - 安装后端 Python 依赖
   - 安装前端依赖并构建
   - 创建桌面快捷方式
4. 安装完成后，桌面会出现 **进销存管理系统** 图标

### 手动安装（如脚本失败）

```cmd
# 1. 克隆代码
cd %USERPROFILE%\Desktop
git clone https://github.com/bingolol/-inventory-system.git inventory-system

# 2. 安装后端依赖
cd inventory-system
pip install -r backend\requirements.txt

# 3. 构建前端
cd frontend
npm install
npm run build
cd ..

# 4. 创建桌面快捷方式
python 创建桌面快捷方式.py
```

## 二、日常使用

双击桌面 **进销存管理系统** 图标即可启动系统。

启动流程：
1. 显示启动画面（带图标和进度条）
2. 后台启动服务
3. 自动打开 pywebview 原生桌面窗口

---

## 三、更新代码

当开发电脑推送了新代码后，生产电脑需要更新：

1. 双击项目目录中的 `一键更新.bat`
2. 脚本会自动完成：
   - 停止正在运行的系统
   - 从 GitHub 拉取最新代码（`git pull`）
   - 更新后端依赖
   - 重新构建前端
   - 刷新桌面快捷方式
3. 更新完成后，双击桌面图标即可使用新版本

### 手动更新（如脚本失败）

```cmd
cd %USERPROFILE%\Desktop\inventory-system

# 停止服务（杀掉 8000 端口进程）
for /f "tokens=5" %p in ('netstat -ano ^| findstr ":8000 " ^| findstr "LISTENING"') do taskkill /F /T /PID %p

# 拉取代码
git pull origin master

# 更新后端依赖
pip install -r backend\requirements.txt

# 重新构建前端
cd frontend
npm install
npm run build
cd ..
```

## 四、开发电脑推送代码

在开发电脑上修改代码后：

```cmd
cd c:\Users\Administrator\Desktop\inventory-system

# 1. 如果修改了前端代码，需要重新构建
cd frontend
npm run build
cd ..

# 2. 提交代码
git add .
git commit -m "描述修改内容"

# 3. 推送到远程仓库
git push origin master
```

推送后，生产电脑运行 `一键更新.bat` 即可同步。

## 六、常见问题

### Q: 启动后白屏
A: 检查 `frontend/dist/index.html` 是否存在。如不存在，在 `frontend/` 目录执行 `npm run build`。

### Q: 后端启动失败
A: 检查 `backend/app.log` 日志文件。常见原因是 Python 依赖未安装完整，执行 `pip install -r backend/requirements.txt`。

### Q: 端口 8000 被占用
A: 启动器会自动清理端口占用。如仍失败，手动执行：
```cmd
for /f "tokens=5" %p in ('netstat -ano ^| findstr ":8000 " ^| findstr "LISTENING"') do taskkill /F /T /PID %p
```

### Q: pywebview 窗口无法打开
A: 确保已安装 `pywebview` 包：`pip install pywebview`。Windows 10/11 自带 WebView2 运行时。

### Q: git pull 冲突
A: 如果生产电脑有本地修改，执行 `git stash` 暂存后再 `git pull`。
